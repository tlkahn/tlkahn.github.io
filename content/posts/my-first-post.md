# Introduction

There are two major schools of thought on the interpretation of probability: one is the frequentist school and the other is the Bayesian school.

Later, we will use the following notation for the observed set: {{< katex >}}
X_{N\times p}=(x_{1},x_{2},\cdots,x_{N})^{T},x_{i}=(x_{i1},x_{i2},\cdots,x_{ip})^{T}
{{< /katex >}}

This notation indicates there are {{< katex >}}N{{< /katex >}} samples, and each sample is a {{< katex >}}p{{< /katex >}}-dimensional vector. Each observation is generated by {{< katex >}}p(x|\theta){{< /katex >}}.

## Frequentist

Frequentists hold that in {{< katex >}}p(x|\theta){{< /katex >}}, {{< katex >}}\theta{{< /katex >}} as a parameter is a constant. For {{< katex >}}N{{< /katex >}} observations, the probability of the observed set is {{< katex >}}p(X|\theta)\mathop{=}\limits _{iid}\prod\limits _{i=1}^{N}p(x_{i}|\theta)){{< /katex >}}. To find the value of {{< katex >}}\theta{{< /katex >}}, we use the maximum log-likelihood (MLE) method:

{{< katex >}}
\theta_{MLE}=\mathop{argmax}\limits _{\theta}\log p(X|\theta)\mathop{=}\limits _{iid}\mathop{argmax}\limits _{\theta}\sum\limits _{i=1}^{N}\log p(x_{i}|\theta)
{{< /katex >}}

## Bayesian

Bayesians believe that {{< katex >}}\theta{{< /katex >}} in {{< katex >}}p(x|\theta){{< /katex >}} is not a constant. This {{< katex >}}\theta{{< /katex >}} follows a preset prior distribution {{< katex >}}\theta\sim p(\theta){{< /katex >}}. So, according to Bayes\' theorem, the parameter\'s posterior distribution based on the observed set can be written as:

{{< katex >}}
p(\theta|X)=\frac{p(X|\theta)\cdot p(\theta)}{p(X)}=\frac{p(X|\theta)\cdot p(\theta)}{\int\limits _{\theta}p(X|\theta)\cdot p(\theta)d\theta} \propto p(X|\theta)\cdot p(\theta)
{{< /katex >}}

To find the value of {{< katex >}}\theta{{< /katex >}}, we need to maximize this parameter\'s posterior distribution using the MAP (Maximum A Posteriori) method:

{{< katex >}}
\theta_{MAP}=\mathop{argmax}\limits _{\theta}p(\theta|X)=\mathop{argmax}\limits _{\theta}p(X|\theta)\cdot p(\theta)
{{< /katex >}}

The second equality is due to the denominator being unrelated to {{< katex >}}\theta{{< /katex >}}.

{{< katex >}}p(X|\theta){{< /katex >}} is called the likelihood and represents our model distribution. After obtaining the posterior distribution of the parameters, we can use it for Bayesian prediction:

{{< katex >}}
p(x_{new}|X)=\int\limits _{\theta}p(x_{new}|\theta)\cdot p(\theta|X)d\theta
{{< /katex >}}

In this integral, the multiplicand is the model (likelihood), and the multiplier is the posterior distribution.

Here\'s the derivation:

1.  Start with the joint probability of observing {{< katex >}}x_{new}{{< /katex >}} and {{< katex >}}\theta{{< /katex >}} given {{< katex >}}X{{< /katex >}}: {{< katex >}}p(x_{new}, \theta|X) = p(x_{new}|\theta, X) \cdot p(\theta|X){{< /katex >}}

2.  To find the marginal probability of observing {{< katex >}}x_{new}{{< /katex >}} given {{< katex >}}X{{< /katex >}}, integrate the joint probability over all possible values of {{< katex >}}\theta{{< /katex >}}: {{< katex >}}p(x_{new}|X) = \int\limits_{\theta} p(x_{new}, \theta|X) d\theta{{< /katex >}}

3.  Substitute the joint probability from step 1 into the integral: {{< katex >}}p(x_{new}|X) = \int\limits_{\theta} [p(x_{new}|\theta, X) \cdot p(\theta|X)] d\theta{{< /katex >}}

4.  Since {{< katex >}}x_{new}{{< /katex >}} is conditionally independent of {{< katex >}}X{{< /katex >}} given {{< katex >}}\theta{{< /katex >}}, we can simplify {{< katex >}}p(x_{new}|\theta, X){{< /katex >}} as {{< katex >}}p(x_{new}|\theta){{< /katex >}}: {{< katex >}}p(x_{new}|X) = \int\limits_{\theta} [p(x_{new}|\theta) \cdot p(\theta|X)] d\theta{{< /katex >}}

The final result is the given formula:

{{< katex >}}p(x_{new}|X) = \int\limits_{\theta} p(x_{new}|\theta) \cdot p(\theta|X) d\theta{{< /katex >}}

# Gaussian Distribution

## 1-_d_ MLE

{{< katex >}}\theta{{< /katex >}} of Gaussian in 1-_d_ MLE

{{< katex display=true class="optional" >}}
\begin{aligned}
&\theta=(\mu,\Sigma)=(\mu,\sigma^{2}), \\
&\theta_{MLE}=\mathop{argmax}\limits_{\theta}\log p(X|\theta)\mathop{=}\limits_{iid}\mathop{argmax}\limits_{\theta}\sum\limits _{i=1}^{N}\log p(x_{i}|\theta)
\end{aligned}
{{< /katex >}}
Probability density function (PDF) of the multivariate Gaussian distribution in {{< katex >}}p{{< /katex >}}-d with mean vector {{< katex >}}\mu{{< /katex >}} and covariance matrix {{< katex >}}\Sigma{{< /katex >}} is written as: {{< katex >}}
p(x|\mu,\Sigma)=\frac{1}{(2\pi)^{p/2}|\Sigma|^{1/2}}e^{-\frac{1}{2}(x-\mu)^{T}\Sigma^{-1}(x-\mu)}
{{< /katex >}} Plugging into MLE, we consider the one-dimensional case: {{< katex >}}
\log p(X|\theta)=\sum\limits _{i=1}^{N}\log p(x_{i}|\theta)=\sum\limits _{i=1}^{N}\log\frac{1}{\sqrt{2\pi}\sigma}\exp(-(x_{i}-\mu)^{2}/2\sigma^{2})
{{< /katex >}} To find {{< katex >}}\mu_{MLE}{{< /katex >}}:

{{< katex >}}
\mu_{MLE}=\mathop{argmax}\limits _{\mu}\log p(X|\theta)=\mathop{argmax}\limits _{\mu}\sum\limits _{i=1}^{N}(x_{i}-\mu)^{2}
{{< /katex >}} {{< katex >}}
\frac{\partial}{\partial\mu}\sum\limits _{i=1}^{N}(x_{i}-\mu)^{2}=0\longrightarrow\mu_{MLE}=\frac{1}{N}\sum\limits _{i=1}^{N}x_{i}
{{< /katex >}} For {{< katex >}}\sigma{{< /katex >}}, we have:

{{< katex display=true class="optional" >}}
\begin{aligned}
\sigma_{MLE}=\mathop{argmax}\limits _{\sigma}\log p(X|\theta)&=\mathop{argmax}\limits _{\sigma}\sum\limits _{i=1}^{N}[-\log\sigma-\frac{1}{2\sigma^{2}}(x_{i}-\mu)^{2}]\\
&=\mathop{argmin}\limits _{\sigma}\sum\limits _{i=1}^{N}[\log\sigma+\frac{1}{2\sigma^{2}}(x_{i}-\mu)^{2}]
\end{aligned}
{{< /katex >}}
{{< katex >}}\therefore{{< /katex >}} {{< katex >}}
\frac{\partial}{\partial\sigma}\sum\limits _{i=1}^{N}[\log\sigma+\frac{1}{2\sigma^{2}}(x_{i}-\mu)^{2}]=0
{{< /katex >}}

Given the log-likelihood function to optimize: {{< katex >}}\sum\limits_{i=1}^{N}[\log\sigma + \frac{1}{2\sigma^2}(x_i - \mu)^2]{{< /katex >}}

1.  Differentiate the log-likelihood function with respect to {{< katex >}}\sigma{{< /katex >}}: {{< katex >}}\frac{\partial}{\partial\sigma}\sum\limits_{i=1}^{N}[\log\sigma + \frac{1}{2\sigma^2}(x_i - \mu)^2]{{< /katex >}}

2.  Apply the sum rule and chain rule for differentiation: {{< katex >}}N \cdot \frac{1}{\sigma} - \sum\limits_{i=1}^{N}\frac{(x_i - \mu)^2}{\sigma^3} = 0{{< /katex >}}

3.  Solve for {{< katex >}}\sigma^2{{< /katex >}}: {{< katex >}}\frac{1}{\sigma} = \frac{1}{N \sigma^3}\sum\limits_{i=1}^{N}(x_i - \mu)^2 \longrightarrow \sigma^2 = \frac{1}{N}\sum\limits_{i=1}^{N}(x_i - \mu)^2{{< /katex >}}

The result is the MLE of the variance: {{< katex >}}\sigma_{MLE}^2 = \frac{1}{N}\sum\limits_{i=1}^{N}(x_i - \mu)^2{{< /katex >}}

Since we have {{< katex >}}\sigma^2_{MLE}{{< /katex >}}, for a unbiased estimate of {{< katex >}}\mathbb{E}_D [\sigma^2_{MLE}]{{< /katex >}}:

{{< katex display=true class="optional" >}}
\begin{aligned}
\mathbb{E}_{\mathcal{D}}[\sigma_{MLE}^{2}]&=\mathbb{E}_{\mathcal{D}}[\frac{1}{N}\sum\limits _{i=1}^{N}(x_{i}-\mu_{MLE})^{2}]=\mathbb{E}_{\mathcal{D}}[\frac{1}{N}\sum\limits _{i=1}^{N}(x_{i}^{2}-2x_{i}\mu_{MLE}+\mu_{MLE}^{2})
\\&=\mathbb{E}_{\mathcal{D}}[\frac{1}{N}\sum\limits _{i=1}^{N}x_{i}^{2}-\mu_{MLE}^{2}]=\mathbb{E}_{\mathcal{D}}[\frac{1}{N}\sum\limits _{i=1}^{N}x_{i}^{2}-\mu^{2}+\mu^{2}-\mu_{MLE}^{2}]\\
&= \mathbb{E}_{\mathcal{D}}[\frac{1}{N}\sum\limits _{i=1}^{N}x_{i}^{2}-\mu^{2}]-\mathbb{E}_{\mathcal{D}}[\mu_{MLE}^{2}-\mu^{2}]=\sigma^{2}-(\mathbb{E}_{\mathcal{D}}[\mu_{MLE}^{2}]-\mu^{2})\\&=\sigma^{2}-(\mathbb{E}_{\mathcal{D}}[\mu_{MLE}^{2}]-\mathbb{E}_{\mathcal{D}}^{2}[\mu_{MLE}])=\sigma^{2}-Var[\mu_{MLE}]\\&=\sigma^{2}-Var[\frac{1}{N}\sum\limits _{i=1}^{N}x_{i}]=\sigma^{2}-\frac{1}{N^{2}}\sum\limits _{i=1}^{N}Var[x_{i}]=\frac{N-1}{N}\sigma^{2}
\end{aligned}
{{< /katex >}}
{{< katex >}}\therefore{{< /katex >}} for {{< katex >}}\hat{\sigma}^{2}{{< /katex >}} a.k.a {{< katex >}}\mathbb{E}_{\mathcal{D}}[\sigma_{MLE}^{2}]{{< /katex >}}:

{{< katex >}}
\hat{\sigma}^{2}=\frac{1}{N-1}\sum\limits _{i=1}^{N}(x_{i}-\mu)^{2}
{{< /katex >}}

## Multivariate Gaussian

{{< katex >}}\because{{< /katex >}}

{{< katex >}}
p(x|\mu,\Sigma)=\frac{1}{(2\pi)^{p/2}|\Sigma|^{1/2}}e^{-\frac{1}{2}(x-\mu)^{T}\Sigma^{-1}(x-\mu)}
{{< /katex >}}

Here, {{< katex >}}x,\mu\in\mathbb{R}^{p},\Sigma\in\mathbb{R}^{p\times p}{{< /katex >}}, where {{< katex >}}\Sigma{{< /katex >}} is the covariance matrix, generally also a semi-positive definite matrix.

First, we deal with the number in the exponent, which can be denoted as the Mahalanobis distance between {{< katex >}}x{{< /katex >}} and {{< katex >}}\mu{{< /katex >}}.

The Mahalanobis distance is a method for measuring the distance between data points, taking into account the covariance structure of the data. For two points {{< katex >}}x{{< /katex >}} and {{< katex >}}y{{< /katex >}}, their Mahalanobis distance is defined as:

{{< katex >}}D_{M}(x, y) = \sqrt{(x - y)^T \Sigma^{-1} (x - y)}{{< /katex >}}

Here, {{< katex >}}\Sigma{{< /katex >}} is the covariance matrix of the data. The difference between Mahalanobis distance and Euclidean distance lies in the fact that the former takes into account the shape of the data distribution, especially the correlation between the data. When the covariance matrix of the data is an identity matrix, the Mahalanobis distance is equal to the Euclidean distance.

The covariance matrix of the data is used to represent the degree of joint variation between different dimensions in the data. For a dataset with {{< katex >}}n{{< /katex >}} samples and {{< katex >}}p{{< /katex >}} features, the covariance matrix {{< katex >}}\Sigma{{< /katex >}} is a {{< katex >}}p \times p{{< /katex >}} matrix, defined as:

{{< katex >}}\Sigma = \frac{1}{n-1} \sum\limits_{i=1}^n (x_i - \bar{x})(x_i - \bar{x})^T{{< /katex >}}

Here, {{< katex >}}x_i{{< /katex >}} is the {{< katex >}}i{{< /katex >}}-th sample point, {{< katex >}}\bar{x}{{< /katex >}} is the sample mean, and {{< katex >}}n{{< /katex >}} is the number of samples.

This equation defines a matrix because each summand {{< katex >}}(x_i - \bar{x})(x_i - \bar{x})^T{{< /katex >}} itself is a matrix. Let\'s explain this in more detail:

Suppose {{< katex >}}x_i{{< /katex >}} and {{< katex >}}\bar{x}{{< /katex >}} are {{< katex >}}p{{< /katex >}}-dimensional column vectors, then {{< katex >}}(x_i - \bar{x}){{< /katex >}} is also a {{< katex >}}p{{< /katex >}}-dimensional column vector. When calculating the outer product {{< katex >}}(x_i - \bar{x})(x_i - \bar{x})^T{{< /katex >}}, we obtain a {{< katex >}}p \times p{{< /katex >}} matrix, where each element is the product of the corresponding elements in {{< katex >}}(x_i - \bar{x}){{< /katex >}}.

Summing these {{< katex >}}p \times p{{< /katex >}} matrices over the sample size {{< katex >}}n{{< /katex >}}, we obtain a cumulative {{< katex >}}p \times p{{< /katex >}} matrix. Finally, the entire sum is divided by {{< katex >}}(n-1){{< /katex >}} to obtain the unbiased covariance matrix {{< katex >}}\Sigma{{< /katex >}}.

The denominator is `n-1`{.verbatim} rather than `n`{.verbatim} because of [Bessel\'s correction](https://en.wikipedia.org/wiki/Bessel%27s_correction), which is used to provide an unbiased estimate of the population variance when working with sample data.

For a symmetric covariance matrix, eigendecomposition can be performed.

{{< katex >}}\Sigma=U\Lambda U^{T}=(u_{1},u_{2},\cdots,u_{p})diag(\lambda_{i})(u_{1},u_{2},\cdots,u_{p})^{T}=\sum\limits _{i=1}^{p}u_{i}\lambda_{i}u_{i}^{T}{{< /katex >}}

{{< katex >}}
\Sigma^{-1}=\sum\limits _{i=1}^{p}u_{i}\frac{1}{\lambda_{i}}u_{i}^{T}
{{< /katex >}}

In this context, {{< katex >}}u_i{{< /katex >}} is the eigenvector of the covariance matrix {{< katex >}}\Sigma{{< /katex >}}. Eigendecomposition decomposes the matrix into a product of eigenvectors {{< katex >}}u_i{{< /katex >}} and eigenvalues {{< katex >}}\lambda_i{{< /katex >}}. {{< katex >}}U{{< /katex >}} is a matrix composed of {{< katex >}}u_i{{< /katex >}} as column vectors corresponding to the eigenvectors. {{< katex >}}\Lambda{{< /katex >}} is a diagonal matrix with elements on the diagonal being the corresponding eigenvalues {{< katex >}}\lambda_i{{< /katex >}}.

{{< katex display=true class="optional" >}}

\begin{aligned}
\Delta&=(x-\mu)^{T}\Sigma^{-1}(x-\mu) \\ &=\sum\limits _{i=1}^{p}(x-\mu)^{T}u_{i}\frac{1}{\lambda_{i}}u_{i}^{T}(x-\mu) \\ &=\sum\limits _{i=1}^{p}\frac{y_{i}^{2}}{\lambda_{i}} \tag{eq:1}
\end{aligned}

{{< /katex >}}
{{< katex >}} y_i = (x-\mu)^T u_i = u_i^T (x-\mu) {{< /katex >}}

We notice that scalar {{< katex >}}y_i{{< /katex >}} is the projection of {{< katex >}}x-\mu{{< /katex >}} onto the eigenvector {{< katex >}}u_i{{< /katex >}} (or the inverse).

Specifically, {{< katex >}}u_i^T(x - \mu){{< /katex >}} calculates the dot product between {{< katex >}}(x - \mu){{< /katex >}} and {{< katex >}}u_i{{< /katex >}}, which gives the component of {{< katex >}}(x - \mu){{< /katex >}} in the direction of {{< katex >}}u_i{{< /katex >}}. This is why we say {{< katex >}}y_i{{< /katex >}} is the projection of {{< katex >}}(x - \mu){{< /katex >}} onto the eigenvector {{< katex >}}u_i{{< /katex >}}.

In Euclidean space, the dot product (also known as the inner product) is a fundamental operation between two vectors, used to calculate their components (or projections) in the same direction. The formula for calculating Euclidean dot product is:

{{< katex >}}u_i^T (x - \mu) = ||u_i|| ||x - \mu|| \cos{\theta}{{< /katex >}}

Where {{< katex >}}||u_i||{{< /katex >}} and {{< katex >}}||x - \mu||{{< /katex >}} represent the lengths of the vectors {{< katex >}}u_i{{< /katex >}} and {{< katex >}}(x - \mu){{< /katex >}}, respectively, and {{< katex >}}\theta{{< /katex >}} represents the angle between them. Since the eigenvector {{< katex >}}u_i{{< /katex >}} is typically normalized to have unit length, i.e., {{< katex >}}||u_i|| = 1{{< /katex >}}, the dot product simplifies to:

{{< katex >}}y_i = u_i^T (x - \mu) = ||x - \mu|| \cos{\theta}{{< /katex >}}

{{< katex >}}\ref{eq:1}{{< /katex >}} represents a generalized concentric ellipses for different values of {{< katex >}}\Delta{{< /katex >}}.

Each term {{< katex >}}\frac{y_{i}^{2}}{\lambda_{i}}{{< /katex >}} represents the contribution of the projection of the vector {{< katex >}}(x - \mu){{< /katex >}} onto the eigenvector {{< katex >}}u_i{{< /katex >}} to the overall shape of the ellipse. The eigenvalues {{< katex >}}\lambda_i{{< /katex >}} determine the scaling of the ellipse along each eigenvector direction.

Now let\'s look at two issues in the practical application of multidimensional Gaussian models:

1.  The number of degrees of freedom for parameters {{< katex >}}\Sigma,\mu{{< /katex >}} is {{< katex >}}O(p^{2}){{< /katex >}}. For high-dimensional data, this leads to too many degrees of freedom.

    Solution: The high degrees of freedom come from {{< katex >}}\Sigma{{< /katex >}} having {{< katex >}}\frac{p(p+1)}{2}{{< /katex >}} free parameters. We can assume that it is a diagonal matrix or even assume that the elements on the diagonal are the same under the isotropic assumption. The former dimension reduction algorithm is Factor Analysis (FA), and the latter is probabilistic PCA (p-PCA).

2.  The second problem is that a single Gaussian distribution is unimodal, and it cannot achieve good results for data distributions with multiple peaks. Solution: Gaussian Mixture Model (GMM).

Next, we will introduce commonly used theorems for multidimensional Gaussian distributions.

Let {{< katex >}}x=(x_1, x_2,\cdots,x_p)^T=(x_{a,m\times 1}, x_{b,n\times1})^T,\mu=(\mu_{a,m\times1}, \mu_{b,n\times1}),\Sigma=\begin{pmatrix}\Sigma_{aa}&\Sigma_{ab}\\\Sigma_{ba}&\Sigma_{bb}\end{pmatrix}{{< /katex >}}

Given: {{< katex >}}x\sim\mathcal{N}(\mu,\Sigma){{< /katex >}}

Given {{< katex >}}x \sim \mathcal{N}(\mu, \Sigma){{< /katex >}}, and {{< katex >}}y \sim Ax + b{{< /katex >}}, then {{< katex >}}y \sim \mathcal{N}(A\mu + b, A\Sigma A^T){{< /katex >}}.

{{< katex >}}\mathbb{E}[y]=\mathbb{E}[Ax+b]=A\mathbb{E}[x]+b=A\mu+b{{< /katex >}} \\
{{< katex >}}Var[y]=Var[Ax+b]=Var[Ax]=A\cdot Var[x]\cdot A^T{{< /katex >}}

Now, using this theorem, we can obtain four quantities {{< katex >}}p(x_a), p(x_b), p(x_a|x_b), p(x_b|x_a){{< /katex >}}

1.  {{< katex >}}x_a = \begin{pmatrix}\mathbb{I}_{m\times m} & \mathbb{O}_{m\times n}\end{pmatrix}\begin{pmatrix}x_a \\ x_b\end{pmatrix}{{< /katex >}}.

    Substituting into the theorem, we get:

    {{< katex display=true class="optional" >}}
    \begin{aligned}
      \mathbb{E}[x_a] = \begin{pmatrix}\mathbb{I} & \mathbb{O}\end{pmatrix}\begin{pmatrix}\mu_a \\
      \mu_b\end{pmatrix} = \mu_a \\
      Var[x_a] = \begin{pmatrix}\mathbb{I} & \mathbb{O}\end{pmatrix}\begin{pmatrix}\Sigma_{aa} & \Sigma_{ab} \\ \Sigma_{ba} & \Sigma_{bb}\end{pmatrix}\begin{pmatrix}\mathbb{I} \\
      \mathbb{O}\end{pmatrix} = \Sigma_{aa}
      \end{aligned}
    {{< /katex >}}
    {{< katex >}}\therefore{{< /katex >}}

    {{< katex >}}x_a \sim \mathcal{N}(\mu_a, \Sigma_{aa}){{< /katex >}}

2.  Similarly, {{< katex >}}x_b \sim \mathcal{N}(\mu_b, \Sigma_{bb}){{< /katex >}}.

3.  For the two conditional probabilities, we introduce three quantities:

    {{< katex >}}
      x_{b\cdot a} = x_b - \Sigma_{ba}\Sigma_{aa}^{-1}x_a\\
      \mu_{b\cdot a} = \mu_b - \Sigma_{ba}\Sigma_{aa}^{-1}\mu_a\\
      \Sigma_{bb\cdot a} = \Sigma_{bb} - \Sigma_{ba}\Sigma_{aa}^{-1}\Sigma_{ab}
      {{< /katex >}}

    In particular, the last equation is called the Schur Complementary of {{< katex >}}\Sigma_{bb}{{< /katex >}}. We can see that: {{< katex >}}
      x_{b\cdot a} = \begin{pmatrix}-\Sigma_{ba}\Sigma_{aa}^{-1} & \mathbb{I}_{n\times n}\end{pmatrix}\begin{pmatrix}x_a \\ x_b\end{pmatrix}
      {{< /katex >}}

    So

    {{< katex display=true class="optional" >}}
    \begin{aligned}
       \mathbb{E}[x_{b\cdot a}] &= \begin{pmatrix}-\Sigma_{ba}\Sigma_{aa}^{-1} & \mathbb{I}_{n\times n}\end{pmatrix}\begin{pmatrix}\mu_a \\ \mu_b\end{pmatrix} = \mu_{b\cdot a}\\
       Var[x_{b\cdot a}] &= \begin{pmatrix}-\Sigma_{ba}\Sigma_{aa}^{-1} & \mathbb{I}_{n\times n}\end{pmatrix}\begin{pmatrix}\Sigma_{aa} & \Sigma_{ab} \\ \Sigma_{ba} & \Sigma_{bb}\end{pmatrix}\begin{pmatrix}-\Sigma_{aa}^{-1}\Sigma_{ba}^T \\ \mathbb{I}_{n\times n}\end{pmatrix} = \Sigma_{bb\cdot a}
      \end{aligned}
    {{< /katex >}}
    Using these three quantities, we can get {{< katex >}}x_b = x_{b\cdot a} + \Sigma_{ba}\Sigma_{aa}^{-1}x_a{{< /katex >}}.

    Therefore: {{< katex >}}
      \mathbb{E}[x_b|x_a] = \mu_{b\cdot a} + \Sigma_{ba}\Sigma_{aa}^{-1}x_a
      {{< /katex >}} {{< katex >}}
      Var[x_b|x_a] = \Sigma_{bb\cdot a}
      {{< /katex >}}

4.  Similarly: {{< katex >}}
      x_{a\cdot b} = x_a - \Sigma_{ab}\Sigma_{bb}^{-1}x_b\\
      \mu_{a\cdot b} = \mu_a - \Sigma_{ab}\Sigma_{bb}^{-1}\mu_b\\
      \Sigma_{aa\cdot b} = \Sigma_{aa} - \Sigma_{ab}\Sigma_{bb}^{-1}\Sigma_{ba}
      {{< /katex >}}

    {{< katex >}}\therefore{{< /katex >}} {{< katex >}}
      \mathbb{E}[x_a|x_b] = \mu_{a\cdot b} + \Sigma_{ab}\Sigma_{bb}^{-1}x_b
      {{< /katex >}} {{< katex >}}
      Var[x_a|x_b] = \Sigma_{aa\cdot b}
      {{< /katex >}}

\begin{example}
Given:
{{< katex >}}p(x) = \mathcal{N}(\mu, \Lambda^{-1}), p(y|x) = \mathcal{N}(Ax + b, L^{-1}){{< /katex >}}, find: {{< katex >}}p(y), p(x|y){{< /katex >}}.
Solution:
Let:
{{< katex >}}y = Ax + b + \epsilon, \epsilon \sim \mathcal{N}(0, L^{-1}){{< /katex >}},
{{< katex >}}\therefore{{< /katex >}}
{{< katex >}}
\begin{aligned}
\mathbb{E}[y] = \mathbb{E}[Ax + b + \epsilon] = A\mu + b
Var[y] = A\Lambda^{-1}A^T + L^{-1}
\end{aligned}
{{< /katex >}}
{{< katex >}}\therefore{{< /katex >}}
{{< katex >}}
  p(y) = \mathcal{N}(A\mu + b, L^{-1} + A\Lambda^{-1}A^T)
{{< /katex >}}
Introduce:
{{< katex >}}z = \begin{pmatrix}x \\ y\end{pmatrix}{{< /katex >}}, we can obtain {{< katex >}}Cov[x, y] = \mathbb{E}[(x - \mathbb{E}[x])(y - \mathbb{E}[y])^T]{{< /katex >}}.
For this covariance, we can directly compute: {{< katex >}}
  \begin{aligned}
  Cov(x, y) &= \mathbb{E}[(x - \mu)(Ax - A\mu + \epsilon)^T] = \mathbb{E}[(x - \mu)(x - \mu)^TA^T] = Var[x]A^T = \Lambda^{-1}A^T
  \end{aligned}
{{< /katex >}}
Note the symmetry of the covariance matrix:
{{< katex >}}p(z) = \mathcal{N}\begin{pmatrix}\mu \\ A\mu + b\end{pmatrix}, \begin{pmatrix}\Lambda^{-1} & \Lambda^{-1}A^T \\ A\Lambda^{-1} & L^{-1} + A\Lambda^{-1}A^T\end{pmatrix}{{< /katex >}}.
{{< katex >}}\therefore{{< /katex >}}
{{< katex >}}
  \mathbb{E}[x|y] = \mu + \Lambda^{-1}A^T(L^{-1} + A\Lambda^{-1}A^T)^{-1}(y - A\mu - b)
  {{< /katex >}}
{{< katex >}}
  Var[x|y] = \Lambda^{-1} - \Lambda^{-1}A^T(L^{-1} + A\Lambda^{-1}A^T)^{-1}A\Lambda^{-1}
{{< /katex >}}
\end{example}
# Linear Regression

\\epigraph{If AI thinks in matrix, so should human.}{anonymous}

Suppose we have dataset {{< katex >}}\mathcal{D}{{< /katex >}}: {{< katex >}}
\mathcal{D}=\{(x_1, y_1),(x_2, y_2),\cdots,(x_N, y_N)\}
{{< /katex >}} We annotate {{< katex >}}X{{< /katex >}} as samples and {{< katex >}}Y{{< /katex >}} as target: {{< katex >}}
X=(x_1,x_2,\cdots,x_N)^T,Y=(y_1,y_2,\cdots,y_N)^T
{{< /katex >}} We are looking at a linear function: {{< katex >}}
f(w)=w^Tx
{{< /katex >}}

## Least squares method

For this problem, we use the squared error defined by the Euclidean norm to define the loss function: {{< katex >}}
L(w) = \sum\limits_{i=1}^N ||w^Tx_i - y_i||^2_2
{{< /katex >}}

Expand it, we get:

{{< katex display=true class="optional" >}}
\begin{aligned}
L(w)&=(w^Tx_1-y_1,\cdots,w^Tx_N-y_N)\cdot (w^Tx_1-y_1,\cdots,w^Tx_N-y_N)^T\\
&=(w^TX^T-Y^T)\cdot (Xw-Y)=w^TX^TXw-Y^TXw-w^TX^TY+Y^TY\\
&=w^TX^TXw-2w^TX^TY+Y^TY
\end{aligned}
{{< /katex >}}
Now minimize {{< katex >}}\hat{w}{{< /katex >}}

{{< katex display=true class="optional" >}}
\begin{aligned}
\hat{w}=\mathop{argmin}\limits_wL(w)&\longrightarrow\frac{\partial}{\partial w}L(w)=0\\
&\longrightarrow2X^TX\hat{w}-2X^TY=0\\
&\longrightarrow \hat{w}=(X^TX)^{-1}X^TY=X^+Y
\end{aligned}
{{< /katex >}}
{{< katex >}}(X^TX)^{-1}X^T{{< /katex >}} is also called the pseudo-inverse. We annote: {{< katex >}}X^+ = (X^TX)^-1X^T{{< /katex >}}

For row full rank or column full rank {{< katex >}}X{{< /katex >}}, the solution can be obtained directly, but for non-full rank sample sets, the singular value decomposition (SVD) method is needed. Apply Singular Value Decomposition (SVD) to {{< katex >}}X{{< /katex >}}, we get {{< katex >}}X=U\Sigma V^T{{< /katex >}}.

To find the pseudoinverse {{< katex >}}X^+{{< /katex >}} for a non-full rank matrix X, we can use the SVD decomposition. We first find the pseudoinverse of the diagonal matrix {{< katex >}}\Sigma{{< /katex >}}. For every non-zero diagonal element {{< katex >}}\sigma_i{{< /katex >}}, its pseudoinverse {{< katex >}}\sigma_i^+{{< /katex >}} is {{< katex >}}\frac{1}{\sigma_i}{{< /katex >}}. Then, the pseudoinverse of {{< katex >}}\Sigma{{< /katex >}}, denoted as {{< katex >}}\Sigma^+{{< /katex >}}, is a diagonal matrix with the reciprocals of the non-zero diagonal elements of {{< katex >}}\Sigma{{< /katex >}}.

Now, using the property of pseudoinverse for orthogonal matrices:

{{< katex >}}U^+ = U^T{{< /katex >}}

{{< katex >}}V^+ = V^T{{< /katex >}}

An orthogonal matrix is a square matrix where its transpose is also its inverse:

{{< katex >}}Q^TQ = QQ^T = I{{< /katex >}}

In the context of SVD, the matrices {{< katex >}}U{{< /katex >}} and {{< katex >}}V{{< /katex >}} are orthogonal matrices. The pseudoinverse, denoted by the superscript +, has the property that for orthogonal matrices, the pseudoinverse is equal to the transpose:

1.  For an orthogonal matrix {{< katex >}}U{{< /katex >}}, we have {{< katex >}}U^TU = UU^T = I{{< /katex >}}, which implies {{< katex >}}U^+ = U^T{{< /katex >}}.
2.  Similarly, for an orthogonal matrix {{< katex >}}V{{< /katex >}}, we have {{< katex >}}V^TV = VV^T = I{{< /katex >}}, which implies {{< katex >}}V^+ = V^T{{< /katex >}}.

The pseudoinverse {{< katex >}}X^+{{< /katex >}} can be computed as:

{{< katex >}}X^+ = (U\Sigma V^T)^+ = V\Sigma^+ U^T{{< /katex >}}

So, we have:

{{< katex >}}X^+ = V\Sigma^{-1}U^T{{< /katex >}}

## MLE with Gaussian noise

For 1-d situation:

{{< katex >}}y=w^Tx+\epsilon,\epsilon\sim\mathcal{N}(0,\sigma^2){{< /katex >}}

{{< katex >}}\therefore y\sim\mathcal{N}(w^Tx,\sigma^2){{< /katex >}}

Plug it into MLE:

{{< katex display=true class="optional" >}}
\begin{aligned}
L(w)=\log p(Y|X,w)&=\log\prod\limits_{i=1}^Np(y_i|x_i,w)\\
&=\sum\limits_{i=1}^N\log(\frac{1}{\sqrt{2\pi\sigma}}e^{-\frac{(y_i-w^Tx_i)^2}{2\sigma^2}})\\
\mathop{argmax}\limits_wL(w)&=\mathop{argmin}\limits_w\sum\limits_{i=1}^N(y_i-w^Tx_i)^2
\end{aligned}
{{< /katex >}}
This expression is the same as the result obtained from the least squares estimation.

## Gaussian weight prior in MAP (Maximum A Posteriori) estimation.

Take the prior distribution {{< katex >}}w \sim \mathcal{N}(0, \sigma_0^2){{< /katex >}}. Then:

{{< katex display=true class="optional" >}}
\begin{aligned}
\hat{w} = \mathop{argmax}\limits_w p(w|Y) &= \mathop{argmax}\limits_w p(Y|w) p(w)  \\
&= \mathop{argmax}\limits_w \log p(Y|w) p(w)  \\
&= \mathop{argmax}\limits_w (\log p(Y|w) + \log p(w))  \\
&= \mathop{argmin}\limits_w [(y - w^Tx)^2 + \frac{\sigma^2}{\sigma_0^2} w^Tw]
\end{aligned}
{{< /katex >}}
Here, we omit {{< katex >}}X{{< /katex >}} as {{< katex >}}p(Y){{< /katex >}} is not related to {{< katex >}}w{{< /katex >}}, and we also use the MLE result for Gaussian distribution from above.

We will see that the existence of the hyperparameter {{< katex >}}\sigma_0{{< /katex >}} corresponds to the Ridge regularization term that will be introduced below. Similarly, if the prior distribution is taken as the Laplace distribution, a result similar to L1 regularization will be obtained.

## Regularization

In practical applications, if the sample size is not much larger than the feature dimension, overfitting may occur. For this situation, we have the following three solutions:

1.  Add more data
2.  Feature selection (reduce feature dimension), such as the PCA algorithm.
3.  Regularization

Regularization generally involves adding a regularization term (representing the penalty for the model\'s complexity) to the loss function (such as the least squares loss introduced above). Below, we introduce two regularization frameworks commonly used in general situations.

{{< katex display=true class="optional" >}}
\begin{aligned}
L1&:\mathop{argmin}\limits_w L(w)+\lambda||w||_1,\lambda > 0 \\
L2&:\mathop{argmin}\limits_w L(w)+\lambda||w||^2_2,\lambda > 0
\end{aligned}
{{< /katex >}}
### L1 Lasso

Caveate: L1 regularization can lead to sparse solutions.

From the perspective of minimizing the loss, the derivative of the L1 term is non-zero near 0 on both the left and right sides, making it easier to obtain a solution of 0.

On the other hand, L1 regularization is equivalent to: {{< katex >}}
\mathop{argmin}\limits_w L(w) \\
s.t. ||w||_1 < C
{{< /katex >}} We have already seen that the squared error loss function is an ellipsoid in the {{< katex >}}w{{< /katex >}} space. Therefore, the solution of the above equation is the tangent point between the ellipsoid and {{< katex >}}||w||_1 = C{{< /katex >}}, making it more likely to be tangent on the coordinate axis. That means that some of the weight coefficients (w) become exactly zero. This property leads to sparsity in the solution, which can be seen as a form of automatic feature selection. However, in certain cases, it might also result in losing some potentially important information from the features whose coefficients become zero.

### L2 Ridge

{{< katex display=true class="optional" >}}
\begin{aligned}
\hat{w}=\mathop{argmin}\limits_wL(w)+\lambda w^Tw&\longrightarrow\frac{\partial}{\partial w}L(w)+2\lambda w=0\\
&\longrightarrow2X^TX\hat{w}-2X^TY+2\lambda \hat w=0\\
&\longrightarrow \hat{w}=(X^TX+\lambda \mathbb{I})^{-1}X^TY
\end{aligned}
{{< /katex >}}
As we can see, this regularization parameter coincides with the previous MAP result. Using the L2 norm for regularization not only allows the model to choose smaller values for {{< katex >}}w{{< /katex >}}, but also avoids the issue of {{< katex >}}X^TX{{< /katex >}} being non-invertible.

## Summary

The linear regression model is the simplest model, but it is a miniature of every machine learning problem. Here, we use the least squares error to obtain a closed-form solution. We also find that when the noise follows a Gaussian distribution, the MLE solution is equivalent to the least squares error. Moreover, after adding a regularization term, the least squares error combined with L2 regularization is equivalent to the MAP estimation under Gaussian noise prior, and with L1 regularization, it is equivalent to the Laplace noise prior.

Traditional machine learning methods more or less contain elements of linear regression models:

1.  Linear models often cannot fit the data well, so there are three strategies to overcome this drawback:
    1.  Transform the feature dimensions, such as polynomial regression models, which add higher-order terms based on linear features.
    2.  Add a non-linear transformation after the linear equation by introducing a non-linear activation function, such as in linear classification models like perceptrons.
    3.  For consistent linear coefficients, we perform multiple transformations so that a single feature is not only affected by a single coefficient, such as in multilayer perceptrons (deep feedforward networks).
2.  Linear regression is linear across the entire sample space. We can modify this limitation by introducing different linear or non-linear functions in different regions, such as in linear spline regression and decision tree models.
3.  Linear regression uses all the samples, but preprocessing the data might yield better learning results (the so-called curse of dimensionality, where high-dimensional data is harder to learn from), such as in PCA algorithms and manifold learning.

# Linear Classifiers

For classification tasks, linear regression models are . However, we can add an activation function to the linear model, which is non-linear. The inverse of the activation function is called the link function. We have two types of linear classification methods:

1.  Hard classification, where we directly need to output the corresponding class of the observation. Representative models of this type include:
    1.  Linear Discriminant Analysis (Fisher Discriminant)
    2.  Perceptron
2.  Soft classification, which generates probabilities for different classes. These algorithms can be divided into two types based on the different probability methods used:
    1.  Discriminative (directly modeling the conditional probability): Logistic Regression

## Hard

### Perceptron

Most intuitively, we can choose an activation function like: {{< katex >}}
sign(a)=\left\{\begin{matrix}+1,a\ge0\\-1,a\leq0\end{matrix}\right.
{{< /katex >}} This way, we can map the result of linear regression to the binary classification result.

Define the loss function as the number of misclassified samples. Start with using the indicator function, but the indicator function is non-differentiable. Therefore, we can define: {{< katex >}}
L(w)=\sum\limits_{x_i\in\mathcal{D}_{wrong}}-y_iw^Tx_i
{{< /katex >}} where {{< katex >}}\mathcal{D}_{wrong}{{< /katex >}} is the set of misclassified samples. In each training step, we use the gradient descent algorithm. The partial derivative of the loss function with respect to {{< katex >}}w{{< /katex >}} is: {{< katex >}}
\frac{\partial}{\partial w}L(w)=\sum\limits_{x_i\in\mathcal{D}_{wrong}}-y_ix_i
{{< /katex >}} However, if there are a large number of samples, the computational complexity is high. In fact, we do not need the absolute direction of the loss function decrease. We only need the expected value of the loss function to decrease, but calculating the expectation requires knowing the true probability distribution. We can only estimate this probability distribution based on the training data sampling (empirical risk):

{{< katex display=true class="optional" >}}

\mathbb{E}_{\mathcal{D}} {\mathbb{E}_{\hat{p}} {[\nabla_wL(w)]}} = \mathbb{E}_{\mathcal D}\left[\frac{1}{N}\sum\limits_{i=1}^N\nabla_wL(w)\right]

{{< /katex >}}
We know that the larger the {{< katex >}}N{{< /katex >}}, the more accurate the sample approximation of the true distribution. However, for a data with a standard deviation of {{< katex >}}\sigma{{< /katex >}}, the determined standard deviation is only inversely proportional to {{< katex >}}\sqrt{N}{{< /katex >}}, while the calculation speed is directly proportional to {{< katex >}}N{{< /katex >}}. Therefore, we can use fewer samples each time, so that the expected loss reduction and calculation speed can be improved. If we use only one misclassified sample each time, we have the following update strategy (based on the Taylor formula, in the negative direction): {{< katex >}}
w^{t+1}\leftarrow w^{t}+\lambda y_ix_i
{{< /katex >}} It is convergent, and using a single observation update can also increase the uncertainty to some extent, thereby reducing the possibility of falling into a local minimum. In larger-scale data, the commonly used method is **mini-batch stochastic gradient descent**.

### LDA

In LDA, our basic idea is to select a direction, project the experimental samples along this direction, and the projected data should satisfy two conditions to achieve better classification:

1.  The distances between experimental samples within the same class are close.
2.  The distances between different classes are larger.

First, let\'s consider the projection. We assume that the original data is a vector {{< katex >}}x{{< /katex >}}, then the projection along the {{< katex >}}w{{< /katex >}} direction is the scalar: {{< katex >}}
z=w^T\cdot x(=|w|\cdot|x|\cos\theta)
{{< /katex >}} For the first point, the samples within the same class are closer. We assume that the number of experimental samples belonging to the two classes are {{< katex >}}N_1{{< /katex >}} and {{< katex >}}N_2{{< /katex >}}, respectively. We use the variance matrix to characterize the overall distribution within each class. Here we use the definition of covariance, denoted by {{< katex >}}S{{< /katex >}} for the original data covariance:

{{< katex display=true class="optional" >}}
\begin{aligned}
C_1:Var_z[C_1]&=\frac{1}{N_1}\sum\limits_{i=1}^{N_1}(z_i-\overline{z_{c1}})(z_i-\overline{z_{c1}})^T\\
&=\frac{1}{N_1}\sum\limits_{i=1}^{N_1}(w^Tx_i-\frac{1}{N_1}\sum\limits_{j=1}^{N_1}w^Tx_j)(w^Tx_i-\frac{1}{N_1}\sum\limits_{j=1}^{N_1}w^Tx_j)^T\\
&=w^T\frac{1}{N_1}\sum\limits_{i=1}^{N_1}(x_i-\overline{x_{c1}})(x_i-\overline{x_{c1}})^Tw\\
&=w^TS_1w\\
C_2:Var_z[C_2]&=\frac{1}{N_2}\sum\limits_{i=1}^{N_2}(z_i-\overline{z_{c2}})(z_i-\overline{z_{c2}})^T\\
&=w^TS_2w
\end{aligned}
{{< /katex >}}
Therefore, the within-class distance can be denoted as:

{{< katex display=true class="optional" >}}
\begin{aligned}
Var_z[C_1]+Var_z[C_2]=w^T(S_1+S_2)w
\end{aligned}
{{< /katex >}}
For the second point, we can use the means of the two classes to represent this distance:

{{< katex display=true class="optional" >}}
\begin{aligned}
(\overline{z_{c1}}-\overline{z_{c2}})^2&=(\frac{1}{N_1}\sum\limits_{i=1}^{N_1}w^Tx_i-\frac{1}{N_2}\sum\limits_{i=1}^{N_2}w^Tx_i)^2\\
&=(w^T(\overline{x_{c1}}-\overline{x_{c2}}))^2\\
&=w^T(\overline{x_{c1}}-\overline{x_{c2}})(\overline{x_{c1}}-\overline{x_{c2}})^Tw
\end{aligned}
{{< /katex >}}
Considering both points, since the covariance is a matrix, we divide these two values to obtain our loss function and maximize this value:

{{< katex display=true class="optional" >}}
\begin{aligned}\hat{w}=\mathop{argmax}\limits_wJ(w)&=\mathop{argmax}\limits_w\frac{(\overline{z_{c1}}-\overline{z_{c2}})^2}{Var_z[C_1]+Var_z[C_2]}\\
&=\mathop{argmax}\limits_w\frac{w^T(\overline{x_{c1}}-\overline{x_{c2}})(\overline{x_{c1}}-\overline{x_{c2}})^Tw}{w^T(S_1+S_2)w}\\
&=\mathop{argmax}\limits_w\frac{w^TS_bw}{w^TS_ww}
\end{aligned}
{{< /katex >}}
In this way, we have combined the loss function with the original dataset and parameters. Next, we will find the partial derivative of this loss function. Note that we actually have no requirement for the absolute value of {{< katex >}}w{{< /katex >}}, only for its direction, so we can solve it with just one equation:

{{< katex display=true class="optional" >}}
\begin{aligned}
&\frac{\partial}{\partial w}J(w)=2S_bw(w^TS_ww)^{-1}-2w^TS_bw(w^TS_ww)^{-2}S_ww=0\\
&\Longrightarrow S_bw(w^TS_ww)=(w^TS_bw)S_ww\\
&\Longrightarrow w\propto S_w^{-1}S_bw=S_w^{-1}(\overline{x_{c1}}-\overline{x_{c2}})(\overline{x_{c1}}-\overline{x_{c2}})^Tw\propto S_w^{-1}(\overline{x_{c1}}-\overline{x_{c2}})
\end{aligned}
{{< /katex >}}
Thus, {{< katex >}}S_w^{-1}(\overline{x_{c1}}-\overline{x_{c2}}){{< /katex >}} is the direction we need to find. Finally, we can normalize it to obtain a unit {{< katex >}}w{{< /katex >}} value.

## Soft

### Logistic regression

Sometimes we just want to obtain the probability of a single class, so we need a function that can output values within the range of {{< katex >}}[0,1]{{< /katex >}}. Considering a binary classification model, we use a discriminative model and hope to model {{< katex >}}p(C|x){{< /katex >}} using Bayes\' theorem: {{< katex >}}
p(C_1|x)=\frac{p(x|C_1)p(C_1)}{p(x|C_1)p(C_1)+p(x|C_2)p(C_2)}
{{< /katex >}} Let {{< katex >}}a=\ln\frac{p(x|C_1)p(C_1)}{p(x|C_2)p(C_2)}{{< /katex >}}, then: {{< katex >}}
p(C_1|x)=\frac{1}{1+\exp(-a)}
{{< /katex >}} The above formula is called the Logistic Sigmoid function, and its parameter represents the logarithm of the ratio of the joint probabilities of the two classes. In the discriminant, we do not care about the specific value of this parameter, and the model assumption is made directly for {{< katex >}}a{{< /katex >}}.

The model assumption of Logistic Regression is: {{< katex >}}
a=w^Tx
{{< /katex >}} Thus, by finding the best value of {{< katex >}}w{{< /katex >}}, we can obtain the best model under this model assumption. The probability discriminative model often uses the maximum likelihood estimation method to determine the parameters.

For a single observation, the probability of obtaining class {{< katex >}}y{{< /katex >}} is (assuming {{< katex >}}C_1=1, C_2=0{{< /katex >}}): {{< katex >}}
p(y|x)=p_1^yp_0^{1-y}
{{< /katex >}}

Then, for {{< katex >}}N{{< /katex >}} independent and identically distributed observations, the Maximum Likelihood Estimation (MLE) is: {{< katex >}}
\hat{w}=\mathop{argmax}_wJ(w)=\mathop{argmax}_w\sum\limits_{i=1}^N(y_i\log p_1+(1-y_i)\log p_0)
{{< /katex >}} Notice that this expression is the negative of the cross-entropy expression multiplied by {{< katex >}}N{{< /katex >}}. The logarithm in MLE also ensures compatibility with the exponential function, thereby obtaining stable gradients in large intervals.

Taking the derivative of this function, we notice that: {{< katex >}}
p_1'=(\frac{1}{1+\exp(-a)})'=p_1(1-p_1)
{{< /katex >}} Then: {{< katex >}}
J'(w)=\sum\limits_{i=1}^Ny_i(1-p_1)x_i-p_1x_i+y_ip_1x_i=\sum\limits_{i=1}^N(y_i-p_1)x_i
{{< /katex >}} Due to the nonlinearity of probability values, this expression cannot be solved directly when placed in the summation symbol. Therefore, in actual training, similar to the perceptron, we can also use batch stochastic gradient ascent (or gradient descent for minimization) with different sizes to obtain the maximum value of this function.

### Gaussian Discrimination Analysis

In generative models, we model the joint probability distribution and then use Maximum A Posteriori (MAP) to obtain the best parameter values. For binary classification, we adopt the following assumption:

1.  {{< katex >}}y\sim Bernoulli(\phi){{< /katex >}}
2.  {{< katex >}}x|y=1\sim\mathcal{N}(\mu_1,\Sigma){{< /katex >}}
3.  {{< katex >}}x|y=0\sim\mathcal{N}(\mu_0,\Sigma){{< /katex >}}

Then, for an independent and identically distributed dataset, the maximum a posteriori probability can be expressed as:

{{< katex display=true class="optional" >}}
\begin{aligned}
\mathop{argmax}_{\phi,\mu_0,\mu_1,\Sigma}\log p(X|Y)p(Y)&=\mathop{argmax}_{\phi,\mu_0,\mu_1,\Sigma}\sum\limits_{i=1}^N (\log p(x_i|y_i)+\log p(y_i))\\
&=\mathop{argmax}_{\phi,\mu_0,\mu_1,\Sigma}\sum\limits_{i=1}^N((1-y_i)\log\mathcal{N}(\mu_0,\Sigma)\\&\quad+y_i\log \mathcal{N}(\mu_1,\Sigma)+y_i\log\phi+(1-y_i)\log(1-\phi))
\end{aligned}
{{< /katex >}}
1.  First, solve for {{< katex >}}\phi{{< /katex >}}, and take the partial derivative of the expression with respect to {{< katex >}}\phi{{< /katex >}}:

    {{< katex display=true class="optional" >}}
    \begin{aligned}\sum\limits_{i=1}^N\frac{y_i}{\phi}+\frac{y_i-1}{1-\phi}=0\\
      \Longrightarrow\phi=\frac{\sum\limits_{i=1}^Ny_i}{N}=\frac{N_1}{N}
      \end{aligned}
    {{< /katex >}}

2.  Solve {{< katex >}}\mu_1{{< /katex >}}

    {{< katex display=true class="optional" >}}
    \begin{aligned}\hat{\mu_1}&=\mathop{argmax}_{\mu_1}\sum\limits_{i=1}^Ny_i\log\mathcal{N}(\mu_1,\Sigma)\\
      &=\mathop{argmin}_{\mu_1}\sum\limits_{i=1}^Ny_i(x_i-\mu_1)^T\Sigma^{-1}(x_i-\mu_1)
      \end{aligned}
    {{< /katex >}}
    {{< katex >}}\because{{< /katex >}} {{< katex >}}
      \sum\limits_{i=1}^Ny_i(x_i-\mu_1)^T\Sigma^{-1}(x_i-\mu_1)=\sum\limits_{i=1}^Ny_ix_i^T\Sigma^{-1}x_i-2y_i\mu_1^T\Sigma^{-1}x_i+y_i\mu_1^T\Sigma^{-1}\mu_1
      {{< /katex >}}

    Taking the derivative of the left side and multiplying by {{< katex >}}\Sigma{{< /katex >}} gives:

    {{< katex display=true class="optional" >}}
    \begin{aligned}\sum\limits_{i=1}^N-2y_i\Sigma^{-1}x_i+2y_i\Sigma^{-1}\mu_1=0\\
      \Longrightarrow\mu_1=\frac{\sum\limits_{i=1}^Ny_ix_i}{\sum\limits_{i=1}^Ny_i}=\frac{\sum\limits_{i=1}^Ny_ix_i}{N_1}
      \end{aligned}
    {{< /katex >}}

3.  Solve {{< katex >}}\mu_0{{< /katex >}}: {{< katex >}}
      \mu_0=\frac{\sum\limits_{i=1}^N(1-y_i)x_i}{N_0}
      {{< /katex >}}

4.  The most difficult part is solving for {{< katex >}}\Sigma{{< /katex >}}. Our model assumes the same covariance matrix for positive and negative examples, although from the above solution, we can see that even if different matrices are used, it will not affect the previous three parameters. First, we have:

    {{< katex display=true class="optional" >}}
    \begin{aligned}
      \sum\limits_{i=1}^N\log\mathcal{N}(\mu,\Sigma)&=\sum\limits_{i=1}^N\log(\frac{1}{(2\pi)^{p/2}|\Sigma|^{1/2}})+(-\frac{1}{2}(x_i-\mu)^T\Sigma^{-1}(x_i-\mu))\\
      &=Const-\frac{1}{2}N\log|\Sigma|-\frac{1}{2}Trace((x_i-\mu)^T\Sigma^{-1}(x_i-\mu))\\
      &=Const-\frac{1}{2}N\log|\Sigma|-\frac{1}{2}Trace((x_i-\mu)(x_i-\mu)^T\Sigma^{-1})\\
      &=Const-\frac{1}{2}N\log|\Sigma|-\frac{1}{2}NTrace(S\Sigma^{-1})
      \end{aligned}
    {{< /katex >}}
    In this expression, we add a trace to the scalar so that the order of the matrices can be exchanged. For the derivative of expressions containing absolute values and traces, we have:

    {{< katex display=true class="optional" >}}
    \begin{aligned}
      \frac{\partial}{\partial A}(|A|)&=|A|A^{-1}\\
      \frac{\partial}{\partial A}Trace(AB)&=B^T
      \end{aligned}
    {{< /katex >}}
    {{< katex >}}\therefore{{< /katex >}}

    {{< katex display=true class="optional" >}}
    \begin{aligned}[\sum\limits_{i=1}^N((1-y_i)\log\mathcal{N}(\mu_0,\Sigma)+y_i\log \mathcal{N}(\mu_1,\Sigma)]'
      \\=Const-\frac{1}{2}N\log|\Sigma|-\frac{1}{2}N_1Trace(S_1\Sigma^{-1})-\frac{1}{2}N_2Trace(S_2\Sigma^{-1})
      \end{aligned}
    {{< /katex >}}
    In which, {{< katex >}}S_1, S_2{{< /katex >}} are the covariance matrices within the two classes of data, so:

    {{< katex display=true class="optional" >}}
    \begin{aligned}N\Sigma^{-1}-N_1S_1^T\Sigma^{-2}-N_2S_2^T\Sigma^{-2}=0
      \\\Longrightarrow\Sigma=\frac{N_1S_1+N_2S_2}{N}
      \end{aligned}
    {{< /katex >}}
    The symmetry of the class covariance matrices is applied here.

Thus, we have obtained all the parameters in our model assumptions using the maximum a posteriori method. According to the model, the joint distribution can be obtained, and hence the conditional distribution for inference can be derived.

### Naive Bayes

The Gaussian discriminant analysis above assumes a Gaussian distribution for the dataset and introduces the Bernoulli distribution as the class prior, thereby obtaining the parameters within these assumptions using maximum a posteriori estimation.

The Naive Bayes model makes assumptions about the relationships between the properties of the data. Generally, we need to obtain the probability value {{< katex >}}p(x|y){{< /katex >}}. Since {{< katex >}}x{{< /katex >}} has {{< katex >}}p{{< /katex >}} dimensions, we need to sample the joint probability of these many dimensions. However, we know that a large number of samples are required to obtain a relatively accurate probability approximation in such a high-dimensional space.

In a general directed probabilistic graphical model, different assumptions are made about the conditional independence relationships between the dimensions of the attributes. The simplest assumption is the conditional independence assumption in the description of the Naive Bayes model: {{< katex >}}
p(x|y)=\prod\limits_{i=1}^pp(x_i|y)
{{< /katex >}} That is: {{< katex >}}
x_i\perp x_j|y,\forall\  i\ne j
{{< /katex >}} Thus, using Bayes\' theorem, for a single observation: {{< katex >}}
p(y|x)=\frac{p(x|y)p(y)}{p(x)}=\frac{\prod\limits_{i=1}^pp(x_i|y)p(y)}{p(x)}
{{< /katex >}} Further assumptions are made for the individual dimensions of conditional probabilities and class priors:

1.  {{< katex >}}x_i{{< /katex >}} is a continuous variable: {{< katex >}}p(x_i|y)=\mathcal{N}(\mu_i,\sigma_i^2){{< /katex >}}
2.  {{< katex >}}x_i{{< /katex >}} is a discrete variable: Categorical distribution: {{< katex >}}p(x_i=i|y)=\theta_i,\sum\limits_{i=1}^K\theta_i=1{{< /katex >}}
3.  {{< katex >}}p(y)=\phi^y(1-\phi)^{1-y}{{< /katex >}}

To estimate these parameters, the MLE method is often used directly on the dataset. Since there is no need to know the relationships between the dimensions, the required amount of data is greatly reduced. After estimating these parameters, they are substituted back into Bayes\' theorem to obtain the posterior distribution of the categories.

## Summary

Classification tasks are divided into two types. For tasks that require directly outputting the class, in the perceptron algorithm, we add a sign function as an activation function to the linear model, which allows us to obtain the class. However, the sign function is not smooth, so we adopt an error-driven approach, introducing {{< katex >}}\sum\limits_{x_i\in\mathcal{D}_{wrong}}-y_iw^Tx_i{{< /katex >}} as the loss function, and then minimize this error using the batch stochastic gradient descent method to obtain the optimal parameter values. In linear discriminant analysis, we consider the linear model as a projection of data points in a certain direction, and use the idea of minimizing within-class variance and maximizing between-class variance to define the loss function. The within-class variance is defined as the sum of the variances of the two classes, and the between-class variance is defined as the distance between the centroids of the two classes. Taking the derivative of the loss function gives the direction of the parameters, which is {{< katex >}}S_w^{-1}(\overline x_{c1}-\overline x_{c2}){{< /katex >}}, where {{< katex >}}S_w{{< /katex >}} is the sum of the variances of the two classes in the original dataset.

Another type of task is to output the probability of classification. For probability models, we have two schemes. The first is the discriminative model, which directly models the conditional probability of the class. By inserting the linear model into the Logistic function, we obtain the Logistic regression model. The probability interpretation here is that the logarithm of the joint probability ratio of the two classes is linear. We define the loss function as the cross-entropy (equivalent to MLE), and taking the derivative of this function gives {{< katex >}}\frac{1}{N}\sum\limits_{i=1}^N(y_i-p_1)x_i{{< /katex >}}. We also use the batch stochastic gradient (ascent) method for optimization. The second is the generative model, which introduces the class prior. In Gaussian discriminant analysis, we assume the distribution of the dataset, where the class prior is a binomial distribution, and the likelihood of each class is a Gaussian distribution. Maximizing the log-likelihood of this joint distribution gives the parameters, {{< katex >}}\frac{\sum\limits_{i=1}^Ny_ix_i}{N_1},\frac{\sum\limits_{i=1}^N(1-y_i)x_i}{N_0},\frac{N_1S_1+N_2S_2}{N},\frac{N_1}{N}{{< /katex >}}. In Naive Bayes, we further assume the dependence between the dimensions of the attributes, and the conditional independence assumption greatly reduces the data requirements.

# Support Vector Machines (SVM)

Support Vector Machines (SVM) play an important role in classification problems, with the main idea being to maximize the margin between two classes. Based on the characteristics of the dataset:

1.  Linearly separable problems, like those previously handled by the perceptron algorithm
2.  Linearly separable, with only a few misclassified points, like the problems addressed by the evolved Pocket algorithm from the perceptron
3.  Nonlinear problems, completely non-separable, such as those tackled by multilayer perceptrons and deep learning developed from the perceptron problem

For these three situations, SVM has the following three methods:

1.  Hard-margin SVM
2.  Soft-margin SVM
3.  Kernel Method

In solving SVMs, Lagrange multiplier method is widely used. First, let\'s start with it.

## Constrained Optimization

Generally, a constrained optimization problem (original problem) can be written as:

{{< katex display=true class="optional" >}}
\begin{aligned}
&\min_{x\in\mathbb{R^p}}f(x)\\
&s.t.\ m_i(x)\le0,i=1,2,\cdots,M\\
&\ \ \ \ \ \ \ \ n_j(x)=0,j=1,2,\cdots,N
\end{aligned}
{{< /katex >}}
Define Lagrange function:

{{< katex >}} L(x,\lambda,\eta) = f(x) + \sum_{i=1}^{M} \lambda_i m_i(x) + \sum_{j=1}^{N} \eta_j n_j(x) {{< /katex >}}

Then, the original problem can be equivalently transformed into an unconstrained form:

{{< katex >}}
\min_x \max_{\lambda, \eta} L(x,\lambda,\eta)
{{< /katex >}}

where {{< katex >}}\lambda{{< /katex >}} is a vector of Lagrange multipliers for inequality constraints and {{< katex >}}eta{{< /katex >}} is a vector of Lagrange multipliers for equality constraints. Note that λ should be non-negative.

This unconstrained form allows us to solve the optimization problem without explicitly considering the constraints, as they are incorporated into the Lagrange function.

This is because, when the inequality constraints of the original problem are satisfied, the maximum value can be obtained when {{< katex >}}\lambda_i=0{{< /katex >}}, which is directly equivalent to the original problem. If the inequality constraints of the original problem are not satisfied, the maximum value becomes {{< katex >}}+\infty{{< /katex >}}. Since the minimum value is required, this situation will not be chosen.

The dual form of this problem is: {{< katex >}}
\max_{\lambda,\eta}\min_{x\in\mathbb{R}^p}L(x,\lambda,\eta)\ s.t.\ \lambda_i\ge0
{{< /katex >}} The dual problem is a maximization problem with respect to {{< katex >}}\lambda, \eta{{< /katex >}}.

{{< katex >}}\because{{< /katex >}} {{< katex >}}
\max_{\lambda_i,\eta_j}\min_{x}L(x,\lambda_i,\eta_j)\le\min_{x}\max_{\lambda_i,\eta_j}L(x,\lambda_i,\eta_j)
{{< /katex >}}

The solution of the dual problem is less than or equal to the original problem, with two situations:

1.  Strong duality: The equality can be achieved
2.  Weak duality: The equality cannot be achieved

For a convex optimization problem, the following theorem holds: If the convex optimization problem satisfies certain conditions, such as the Slater condition, then it and its dual problem satisfy a strong duality relationship. Let the problem\'s domain be defined as: {{< katex >}}\mathcal{D}=domf(x)\cap dom m_i(x)\cap domn_j(x){{< /katex >}}. The Slater condition is: {{< katex >}}
  \exists \hat{x} \in Relint \mathcal{D}\ such\ that\ \forall i=1,2,\cdots,M, m_i(x) < 0
  {{< /katex >}} where {{< katex >}}Relint{{< /katex >}} represents the relative interior (interior not containing the boundary).

For most convex optimization problems, the Slater condition holds; relax the Slater condition: if there are K affine functions among the M inequality constraints, then it is sufficient for the remaining functions to satisfy the Slater condition.

The above introduced the duality relationship between the original problem and the dual problem, but in practice, it is necessary to solve for the parameters. The solution method uses the KKT (Karush-Kuhn-Tucker) conditions:

The KKT conditions and the strong duality relationship are equivalent. The KKT conditions for the optimal solution are:

1.  Feasible domain:

    {{< katex display=true class="optional" >}}
    \begin{aligned}
       m_i(x^*)\le0\\
       n_j(x^*)=0\\
       \lambda^*\ge0
       \end{aligned}
    {{< /katex >}}

2.  Complementary slackness {{< katex >}}\lambda^*m_i(x^*)=0,\forall m_i{{< /katex >}}. The optimal value of the dual problem is {{< katex >}}d^*{{< /katex >}}, and the original problem is {{< katex >}}p^*{{< /katex >}}

    {{< katex display=true class="optional" >}}
    \begin{aligned}
       d^*&=\max_{\lambda,\eta}g(\lambda,\eta)=g(\lambda^*,\eta^*)\\
       &=\min_{x}L(x,\lambda^*,\eta^*)\\
       &\le L(x^*,\lambda^*,\eta^*)\\
       &=f(x^*)+\sum\limits_{i=1}^M\lambda^*m_i(x^*)\\
       &\le f(x^*)=p^*
       \end{aligned}
    {{< /katex >}}

To satisfy the equality, both inequalities must hold. Therefore, for the first inequality, the gradient must be 0; for the second inequality, the complementary slackness condition must be satisfied.

1.  Gradient is 0: {{< katex >}}\frac{\partial L(x,\lambda^*,\eta^*)}{\partial x}|_{x=x^*}=0{{< /katex >}}

## Hard-margin SVM

Support Vector Machine (SVM) is also a kind of hard classification model. In the previous perceptron model, we added a sign function on top of the linear model. From a geometric intuition, we can see that if the two classes are well separated, there will actually be infinitely many lines that can separate them. In SVM, we introduce the concept of maximizing the margin, where the margin refers to the minimum distance between the data and the dividing line. Maximizing this value reflects the tendency of our model.

The separating hyperplane can be written as: {{< katex >}}
0=w^Tx+b
{{< /katex >}} Then, maximize the margin (subject to the constraints of the classification task): {{< katex >}}
\mathop{argmax}_{w,b}[\min_i\frac{|w^Tx_i+b|}{||w||}]\ s.t.\ y_i(w^Tx_i+b)>0\\
\Longrightarrow\mathop{argmax}_{w,b}[\min_i\frac{y_i(w^Tx_i+b)}{||w||}]\ s.t.\ y_i(w^Tx_i+b)>0
{{< /katex >}} For this constraint {{< katex >}}y_i(w^Tx_i+b)>0{{< /katex >}}, we can fix {{< katex >}}\min y_i(w^Tx_i+b)=1>0{{< /katex >}}, since scaling the coefficients of the hyperplane separating the two classes does not change the plane. This is equivalent to constraining the coefficients of the hyperplane. The simplified expression can be represented as: {{< katex >}}
\mathop{argmin}_{w,b}\frac{1}{2}w^Tw\ s.t.\ \min_iy_i(w^Tx_i+b)=1\\
\Rightarrow\mathop{argmin}_{w,b}\frac{1}{2}w^Tw\ s.t.\ y_i(w^Tx_i+b)\ge1,i=1,2,\cdots,N
{{< /katex >}} This is a convex optimization problem with {{< katex >}}N{{< /katex >}} constraints, and there are many software tools for solving such problems.

However, if the sample size or dimension is very high, solving the problem directly can be difficult or even infeasible, so further processing is needed. Introduce the Lagrange function: {{< katex >}}
L(w,b,\lambda)=\frac{1}{2}w^Tw+\sum\limits_{i=1}^N\lambda_i(1-y_i(w^Tx_i+b))
{{< /katex >}} The original problem is equivalent to: {{< katex >}}
\mathop{argmin}_{w,b}\max_{\lambda}L(w,b,\lambda_i)\ s.t.\ \lambda_i\ge0
{{< /katex >}} We swap the minimum and maximum symbols to obtain the dual problem: {{< katex >}}
\max_{\lambda_i}\min_{w,b}L(w,b,\lambda_i)\ s.t.\ \lambda_i\ge0
{{< /katex >}} Since the inequality constraint is an affine function, the dual problem is equivalent to the original problem:

-   {{< katex >}}b{{< /katex >}}:{{< katex >}}\frac{\partial}{\partial b}L=0\Rightarrow\sum\limits_{i=1}^N\lambda_iy_i=0{{< /katex >}}

-   {{< katex >}}w{{< /katex >}}:plug in {{< katex >}}b{{< /katex >}} {{< katex >}}
     L(w,b,\lambda_i)=\frac{1}{2}w^Tw+\sum\limits_{i=1}^N\lambda_i(1-y_iw^Tx_i-y_ib)=\frac{1}{2}w^Tw+\sum\limits_{i=1}^N\lambda_i-\sum\limits_{i=1}^N\lambda_iy_iw^Tx_i
     {{< /katex >}} {{< katex >}}\therefore{{< /katex >}} {{< katex >}}
     \frac{\partial}{\partial w}L=0\Rightarrow w=\sum\limits_{i=1}^N\lambda_iy_ix_i
     {{< /katex >}}

-   Plug in both parameters {{< katex >}}
     L(w,b,\lambda_i)=-\frac{1}{2}\sum\limits_{i=1}^N\sum\limits_{j=1}^N\lambda_i\lambda_jy_iy_jx_i^Tx_j+\sum\limits_{i=1}^N\lambda_i
     {{< /katex >}}

Therefore, the dual problem is: {{< katex >}}
\max_{\lambda}-\frac{1}{2}\sum\limits_{i=1}^N\sum\limits_{j=1}^N\lambda_i\lambda_jy_iy_jx_i^Tx_j+\sum\limits_{i=1}^N\lambda_i,\ s.t.\ \lambda_i\ge0
{{< /katex >}} The parameters of the hyperplane can be obtained from the KKT conditions:

The necessary and sufficient conditions for the strong duality relationship between the original problem and the dual problem are that they satisfy the KKT conditions:

{{< katex display=true class="optional" >}}
\begin{aligned}
  &\frac{\partial L}{\partial w}=0,\frac{\partial L}{\partial b}=0
  \\&\lambda_k(1-y_k(w^Tx_k+b))=0(slackness\ complementary)\\
  &\lambda_i\ge0\\
  &1-y_i(w^Tx_i+b)\le0
  \end{aligned}
{{< /katex >}}
Based on these conditions, we can obtain the corresponding optimal parameters:

{{< katex display=true class="optional" >}}
\begin{aligned}
\hat{w}&=\sum\limits_{i=1}^N\lambda_iy_ix_i\\
\hat{b}&=y_k-w^Tx_k=y_k-\sum\limits_{i=1}^N\lambda_iy_ix_i^Tx_k, \\
&\exists k,1-y_k(w^Tx_k+b)=0
\end{aligned}
{{< /katex >}}
Thus, the parameter {{< katex >}}w{{< /katex >}} of the hyperplane is a linear combination of data points, and the final parameter values are the linear combination of some vectors that satisfy {{< katex >}}y_i(w^Tx_i+b)=1{{< /katex >}} (given by the complementary slackness condition). These vectors are also called support vectors.

## Soft-margin SVM

Hard-margin SVM is only solvable for separable data. If the data is not separable, our basic idea is to introduce the possibility of misclassification into the loss function. The number of misclassifications can be written as: {{< katex >}}
error=\sum\limits_{i=1}^N\mathbb{I}\{y_i(w^Tx_i+b)<1\}
{{< /katex >}} This function is discontinuous, so we can rewrite it as: {{< katex >}}
error=\sum\limits_{i=1}^N\max\{0,1-y_i(w^Tx_i+b)\}
{{< /katex >}} The term inside the summation is called the Hinge Function.

By adding this error term into the hard-margin SVM, we have:

{{< katex display=true class="optional" >}}

\begin{aligned}
&\mathop{argmin}_{w,b}\frac{1}{2}w^Tw+C\sum\limits_{i=1}^N\max\{0,1-y_i(w^Tx_i+b)\} \\
&\textrm{s.t.} \quad y_i(w^Tx_i+b)\ge1-\xi_i,i=1,2,\cdots,N
\end{aligned}

{{< /katex >}}
In this expression, the constant {{< katex >}}C{{< /katex >}} can be considered as the allowed error level. To further eliminate the {{< katex >}}\max{{< /katex >}} symbol, for each observation in the dataset, we can assume that most of them satisfy the constraint, but some of them violate the constraint. Therefore, this part of the constraint becomes {{< katex >}}y_i(w^Tx+b)\ge1-\xi_i{{< /katex >}}, where {{< katex >}}\xi_i=1-y_i(w^Tx_i+b){{< /katex >}}. Further simplification gives: {{< katex >}}
\mathop{argmin}_{w,b}\frac{1}{2}w^Tw+C\sum\limits_{i=1}^N\xi_i\ s.t.\ y_i(w^Tx_i+b)\ge1-\xi_i,\xi_i\ge0,i=1,2,\cdots,N
{{< /katex >}}

## Kernel Method

Kernel methods can be applied to many problems. In classification problems, for strictly non-separable problems, we introduce a feature transformation function to transform the original non-separable dataset into a separable dataset, and then apply the existing model. Often, when transforming a low-dimensional dataset into a high-dimensional dataset, the data becomes separable (the data becomes sparser):

High-dimensional spaces are more likely to be linearly separable than low-dimensional spaces.

When applied to SVM, we observe the dual problem of SVM: {{< katex >}}
\max_{\lambda}-\frac{1}{2}\sum\limits_{i=1}^N\sum\limits_{j=1}^N\lambda_i\lambda_jy_iy_jx_i^Tx_j+\sum\limits_{i=1}^N\lambda_i,\ s.t.\ \lambda_i\ge0
{{< /katex >}} When solving, we need to find the inner product, so after the feature transformation of non-separable data, we need to find the inner product of the transformed data. It is often difficult to find the inner product of the transformation function. Hence, we directly introduce a transformation function of the inner product: {{< katex >}}
\forall x,x'\in\mathcal{X},\exists\phi\in\mathcal{H}:x\rightarrow z\ s.t.\ k(x,x')=\phi(x)^T\phi(x)
{{< /katex >}} The function {{< katex >}}k(x,x'){{< /katex >}} is called a positive definite kernel function, where {{< katex >}}\mathcal{H}{{< /katex >}} is a Hilbert space (a complete linear inner product space). If we remove the inner product condition, we simply call it a kernel function.

{{< katex >}}k(x,x')=\exp(-\frac{(x-x')^2}{2\sigma^2}){{< /katex >}} is a kernel function.

{{< katex display=true class="optional" >}}

\begin{aligned}
\exp(-\frac{(x-x\prime)^2}{2\sigma^2})&=\exp(-\frac{x^2}{2\sigma^2})\exp(\frac{xx\prime}{\sigma^2})\exp(-\frac{x\prime^2}{2\sigma^2})\\
&=\exp(-\frac{x^2}{2\sigma^2})\sum\limits_{n=0}^{+\inf}\frac{x^nx\prime^n}{\sigma^{2n}n!}\exp(-\frac{x\prime^2}{2\sigma^2})\\
&=\exp(-\frac{x^2}{2\sigma^2})\varphi(x)\varphi(x\prime)\exp(-\frac{x\prime^2}{2\sigma^2})\\&=\phi(x)\phi(x\prime)
\end{aligned}

{{< /katex >}}
A positive definite kernel function has the following equivalent definitions:

If the kernel function satisfies:

1.  Symmetry
2.  Positive definiteness

Then this kernel function is a positive definite kernel function.

Proof:

1.  Symmetry {{< katex >}}\Leftrightarrow{{< /katex >}} {{< katex >}}k(x,z)=k(z,x){{< /katex >}}, which obviously satisfies the definition of an inner product.
2.  Positive definiteness {{< katex >}}\Leftrightarrow{{< /katex >}} {{< katex >}}\forall N,x_1,x_2,\cdots,x_N\in\mathcal{X}{{< /katex >}}, the corresponding Gram Matrix {{< katex >}}K=[k(x_i,x_j)]{{< /katex >}} is positive semi-definite.

To prove: {{< katex >}}k(x,z)=\phi(x)^T\phi(z)\Leftrightarrow K{{< /katex >}} is positive semi-definite and symmetric.

1.  {{< katex >}}\Rightarrow{{< /katex >}}: Firstly, symmetry is obvious. For positive definiteness: {{< katex >}}
      K=\begin{pmatrix}k(x_1,x_2)&\cdots&k(x_1,x_N)\\\vdots&\vdots&\vdots\\k(x_N,x_1)&\cdots&k(x_N,x_N)\end{pmatrix}
      {{< /katex >}} Take any {{< katex >}}\alpha\in\mathbb{R}^N{{< /katex >}}, we need to prove {{< katex >}}\alpha^TK\alpha\ge0{{< /katex >}}: {{< katex >}}
      \alpha^TK\alpha=\sum\limits_{i,j}\alpha_i\alpha_jK_{ij}=\sum\limits_{i,j}\alpha_i\phi^T(x_i)\phi(x_j)\alpha_j=\sum\limits_{i}\alpha_i\phi^T(x_i)\sum\limits_{j}\alpha_j\phi(x_j)
      {{< /katex >}} This expression is in the form of an inner product. Hilbert space satisfies linearity, so the proof of positive definiteness is complete.

2.  {{< katex >}}\Leftarrow{{< /katex >}}: Decompose {{< katex >}}K{{< /katex >}}, for the symmetric matrix {{< katex >}}K=V\Lambda V^T{{< /katex >}}, then let {{< katex >}}\phi(x_i)=\sqrt{\lambda_i}V_i{{< /katex >}}, where {{< katex >}}V_i{{< /katex >}} is the eigenvector, and we construct {{< katex >}}k(x,z)=\sqrt{\lambda_i\lambda_j}V_i^TV_j{{< /katex >}}.

## Summary

For a long time, classification problems relied on SVM. For strictly separable datasets, Hard-margin SVM selects a hyperplane that maximizes the distance to all data points. A constraint is applied to this plane, fixing {{< katex >}}y_i(w^Tx_i+b)=1{{< /katex >}}, resulting in a convex optimization problem with all constraint conditions as affine functions. This satisfies the Slater condition, and the problem is transformed into a dual problem, obtaining an equivalent solution and constraint parameters:

{{< katex >}}
\max_{\lambda}-\frac{1}{2}\sum\limits_{i=1}^N\sum\limits_{j=1}^N\lambda_i\lambda_jy_iy_jx_i^Tx_j+\sum\limits_{i=1}^N\lambda_i,\ s.t.\ \lambda_i\ge0
{{< /katex >}}

The required hyperplane parameters are solved using the KKT conditions of the strong dual problem:

{{< katex display=true class="optional" >}}
\begin{aligned}
&\frac{\partial L}{\partial w}=0,\frac{\partial L}{\partial b}=0
\\&\lambda_k(1-y_k(w^Tx_k+b))=0(slackness\ complementary)\\
&\lambda_i\ge0\\
&1-y_i(w^Tx_i+b)\le0
\end{aligned}
{{< /katex >}}

The solution is:

{{< katex display=true class="optional">}}
\begin{aligned}
\hat{w}=\sum\limits_{i=1}^N\lambda_iy_ix_i\\
\hat{b}=y_k-w^Tx_k=y_k-\sum\limits_{i=1}^N\lambda_iy_ix_i^Tx_k \\
\exists k,1-y_k(w^Tx_k+b)=0
\end{aligned}
{{< /katex >}}

When allowing for some errors, an error term can be added to the Hard-margin SVM. The Hinge Function represents the size of the error term, resulting in: {{< katex >}}
\mathop{argmin}_{w,b}\frac{1}{2}w^Tw+C\sum\limits_{i=1}^N\xi_i\ s.t.\ y_i(w^Tx_i+b)\ge1-\xi_i,\xi_i\ge0,i=1,2,\cdots,N
{{< /katex >}}

For completely non-separable problems, we use feature transformation. In SVM, we introduce a positive definite kernel function to directly transform the inner product. As long as this transformation satisfies symmetry and positive definiteness, it can be used as a kernel function.

# Exponential Distribution

The exponential family is a class of distributions that includes Gaussian distribution, Bernoulli distribution, binomial distribution, Poisson distribution, Beta distribution, Dirichlet distribution, Gamma distribution, and a series of other distributions. Exponential family distributions can be written in a unified form:

{{< katex >}}
p(x|\eta)=h(x)\exp(\eta^T\phi(x)-A(\eta))=\frac{1}{\exp(A(\eta))}h(x)\exp(\eta^T\phi(x))
{{< /katex >}}

{{< katex >}}\eta{{< /katex >}} is the parameter vector, and {{< katex >}}A(\eta){{< /katex >}} is the log partition function (normalization factor).

In this expression, {{< katex >}}\phi(x){{< /katex >}} is called the sufficient statistic, containing all the information of the sample set, such as the mean and variance in the Gaussian distribution. Sufficient statistics have applications in online learning; for a dataset, it is only necessary to record the sufficient statistics of the samples.

For a model distribution assumption (likelihood), we often need to find a conjugate prior in the solution, so that the prior and posterior forms are the same. For example, if the likelihood is a binomial distribution, the prior can be chosen as a Beta distribution, and the posterior is also a Beta distribution. Exponential family distributions often have conjugate properties, making model selection and inference much more convenient.

The properties of conjugate priors facilitate computation, and at the same time, exponential family distributions satisfy the idea of maximum entropy (uninformative prior). That is, the distribution derived from the empirical distribution using the principle of maximum entropy is the exponential family distribution.

Noticing that the expression of the exponential family distribution is similar to the linear model, in fact, the exponential family distribution naturally derives the generalized linear model: {{< katex >}}
y=f(w^Tx)\\
y|x\sim Exp Family
{{< /katex >}} In more complex probabilistic graphical models, such as in undirected graphical models like Restricted Boltzmann Machines, exponential family distributions also play an important role.

In inference algorithms, such as variational inference, exponential family distributions greatly simplify the computation.

## 1-_d_ Gaussian distribution

1-{{< katex >}}d{{< /katex >}} Gaussian distribution can be written as: {{< katex >}}
p(x|\theta)=\frac{1}{\sqrt{2\pi}\sigma}\exp(-\frac{(x-\mu)^2}{2\sigma^2})
{{< /katex >}}

Now transform the above:

{{< katex display=true class="optional" >}}

\begin{aligned}
&\frac{1}{\sqrt{2\pi\sigma^2}}\exp(-\frac{1}{2\sigma^2}(x^2-2\mu x+\mu^2))\\
&=\exp(\log(2\pi\sigma^2)^{-1/2})\exp(-\frac{1}{2\sigma^2}\begin{pmatrix}-2\mu&1\end{pmatrix}\begin{pmatrix}x\\x^2\end{pmatrix}-\frac{\mu^2}{2\sigma^2})
\end{aligned}

{{< /katex >}}
{{< katex >}}\therefore{{< /katex >}} {{< katex >}}\eta=\begin{pmatrix}\frac{\mu}{\sigma^2}\\-\frac{1}{2\sigma^2}\end{pmatrix}=\begin{pmatrix}\eta_1\\\eta_2\end{pmatrix}{{< /katex >}}, {{< katex >}}A(\eta)=-\frac{\eta_1^2}{4\eta_2}+\frac{1}{2}\log(-\frac{\pi}{\eta_2}){{< /katex >}}

## Sufficient statistics and log partition functions

Integrate the probability density function:

{{< katex display=true class="optional" >}}
\begin{aligned}
\exp(A(\eta))&=\int h(x)\exp(\eta^T\phi(x))dx
\end{aligned}
{{< /katex >}}
Take the derivative with respect to the parameter on both sides:

{{< katex display=true class="optional" >}}
\begin{aligned}
\exp(A(\eta))A'(\eta)&=\int h(x)\exp(\eta^T\phi(x))\phi(x)dx\\
\Longrightarrow A'(\eta)&=\mathbb{E}_{p(x|\eta)}[\phi(x)]
\end{aligned}
{{< /katex >}}
Similarly: {{< katex >}}
A''(\eta)=Var_{p(x|\eta)}[\phi(x)]
{{< /katex >}}

Since the variance is positive, {{< katex >}}A(\eta){{< /katex >}} must be a convex function.

## Sufficient statistics and MLE

For an independently and identically distributed ({{< katex >}}iid{{< /katex >}}) dataset {{< katex >}}\mathcal{D}=\{x_1,x_2,\cdots,x_N\}{{< /katex >}}

{{< katex display=true class="optional" >}}
\begin{aligned}
\eta_{MLE}&=\mathop{argmax}_\eta\sum\limits_{i=1}^N\log p(x_i|\eta)\\
&=\mathop{argmax}_\eta\sum\limits_{i=1}^N(\eta^T\phi(x_i)-A(\eta))\\
&\Longrightarrow A'(\eta_{MLE})=\frac{1}{N}\sum\limits_{i=1}^N\phi(x_i)
\end{aligned}
{{< /katex >}}
From this, we can see that in order to estimate the parameters, it is sufficient to know the sufficient statistics.

## Maximum entrophy

The information entropy is denoted as: {{< katex >}}
Entropy=\int-p(x)\log(p(x))dx
{{< /katex >}}

In general, for a completely random variable (equally likely), the information entropy is maximized.

Our assumption is based on the principle of maximum entropy. Assuming the data follows a discrete distribution with probability of {{< katex >}}k{{< /katex >}} features being {{< katex >}}p_k{{< /katex >}}, the principle of maximum entropy can be formulated as: {{< katex >}}
  \max\{H(p)\}=\min\{\sum\limits_{k=1}^Kp_k\log p_k\}\ s.t.\ \sum\limits_{k=1}^Kp_k=1
  {{< /katex >}}

Using the Lagrange multiplier method: {{< katex >}}
  L(p,\lambda)=\sum\limits_{k=1}^Kp_k\log p_k+\lambda(1-\sum\limits_{k=1}^Kp_k)
  {{< /katex >}}

Hence, we can obtain: {{< katex >}}
  p_1=p_2=\cdots=p_K=\frac{1}{K}
  {{< /katex >}}

Therefore, the entropy is maximized when all probabilities are equal.

For a dataset {{< katex >}}\mathcal{D}{{< /katex >}}, the empirical distribution on this dataset is {{< katex >}}\hat{p}(x)=\frac{Count(x)}{N}{{< /katex >}}. In practice, it is impossible to satisfy all empirical probabilities being equal. Therefore, we need to add the constraint of this empirical distribution in the principle of maximum entropy.

For any function, the empirical expectation of the empirical distribution can be obtained as: {{< katex >}}
\mathbb{E}_{\hat{p}}[f(x)]=\Delta
{{< /katex >}}

{{< katex >}}\therefore
\max\{H(p)\}=\min\{\sum\limits_{k=1}^Np_k\log p_k\}\ s.t.\ \sum\limits_{k=1}^Np_k=1,\mathbb{E}_p[f(x)]=\Delta
{{< /katex >}}

The Lagrange function is: {{< katex >}}
L(p,\lambda_0,\lambda)=\sum\limits_{k=1}^Np_k\log p_k+\lambda_0(1-\sum\limits_{k=1}^Np_k)+\lambda^T(\Delta-\mathbb{E}_p[f(x)])
{{< /katex >}}

Taking the derivative, we get:

{{< katex >}}
\frac{\partial}{\partial p(x)}L=\sum\limits_{k=1}^N(\log p(x)+1)-\sum\limits_{k=1}^N\lambda_0-\sum\limits_{k=1}^N\lambda^Tf(x)\\
\Longrightarrow\sum\limits_{k=1}^N\log p(x)+1-\lambda_0-\lambda^Tf(x)=0
{{< /katex >}}

Since the dataset is arbitrary, summing over the dataset also means that every term in the sum is 0: {{< katex >}}
p(x)=\exp(\lambda^Tf(x)+\lambda_0-1)
{{< /katex >}} This is the exponential family distribution.

# Expectation--Maximization Algorithm

The purpose of the Expectation-Maximization (EM) algorithm is to solve the parameter estimation (maximum likelihood estimation) for mixture models with latent variables. The MLE for the parameter estimation of {{< katex >}}p(x|\theta){{< /katex >}} is denoted as: {{< katex >}}\theta_{MLE}=\mathop{argmax}\limits_\theta\log p(x|\theta){{< /katex >}}. The EM algorithm solves this problem using an iterative method:

{{< katex >}}\theta^{\text{t}+1} = \arg\max_{\theta} \int_{z} \log [p(x,z|\theta)] p(z|x,\theta^{\text{t}}) dz = \mathbb{E}_{z|x,\theta^{\text{t}}}[\log p(x,z|\theta)]{{< /katex >}}

This formula includes two iterative steps:

1.  E step: Calculate the expectation of {{< katex >}}\log p(x,z|\theta){{< /katex >}} under the probability distribution {{< katex >}}p(z|x,\theta^t){{< /katex >}}
2.  M step: Calculate the parameters that maximize this expectation to obtain the input for the next EM step

{{< katex >}}\log p(x|\theta^t)\le\log p(x|\theta^{t+1}){{< /katex >}}

{{< katex >}}\log p(x|\theta)=\log p(z,x|\theta)-\log p(z|x,\theta){{< /katex >}}

Integrate both sides:

\begin{aligned}
  Left:&\int_zp(z|x,\theta^t)\log p(x|\theta)dz=\log p(x|\theta) \\
  Right:&\int_zp(z|x,\theta^t)\log p(x,z|\theta)dz-\int_zp(z|x,\theta^t)\log p(z|x,\theta)dz=Q(\theta,\theta^t)-H(\theta,\theta^t)
\end{aligned}

{{< katex >}}\therefore
  \log p(x|\theta)=Q(\theta,\theta^t)-H(\theta,\theta^t)
{{< /katex >}}

\begin{aligned}
\because Q(\theta,\theta^t)&=\int_zp(z|x,\theta^t)\log p(x,z|\theta)dz \\
\theta^{t+1}&=\mathop{argmax}\limits_{\theta}\int_z\log [p(x,z|\theta)]p(z|x,\theta^t)dz \\
\therefore Q(\theta^{t+1},\theta^t)&\ge Q(\theta^t,\theta^t)
\end{aligned}

To have {{< katex >}}\log p(x|\theta^t)\le\log p(x|\theta^{t+1}){{< /katex >}}

We need {{< katex >}}H(\theta^t,\theta^t)\ge H(\theta^{t+1},\theta^t){{< /katex >}}

\begin{aligned}
H(\theta^{t+1},\theta^t)-H(\theta^{t},\theta^t)&=\int_zp(z|x,\theta^{t})\log p(z|x,\theta^{t+1})dz-\int_zp(z|x,\theta^t)\log p(z|x,\theta^{t})dz\\
&=\int_zp(z|x,\theta^t)\log\frac{p(z|x,\theta^{t+1})}{p(z|x,\theta^t)} \\
&=-KL(p(z|x,\theta^t),p(z|x,\theta^{t+1}))\le0
\end{aligned}

Combining everything we have got: {{< katex >}}
  \log p(x|\theta^t)\le\log p(x|\theta^{t+1})
  {{< /katex >}}

Based on the proof above, we see that the likelihood function increases at each step. Furthermore, we look at how the formula in the EM iteration process is derived:

{{< katex >}}
\log p(x|\theta)=\log p(z,x|\theta)-\log p(z|x,\theta)=\log \frac{p(z,x|\theta)}{q(z)}-\log \frac{p(z|x,\theta)}{q(z)}
{{< /katex >}}

Take the expectation {{< katex >}}\mathbb{E}_{q(z)}{{< /katex >}} on both sides:

{{< katex display=true class="optional" >}}
\begin{aligned}
&Left:\int_zq(z)\log p(x|\theta)dz=\log p(x|\theta)\\
&Right:\int_zq(z)\log \frac{p(z,x|\theta)}{q(z)}dz-\int_zq(z)\log \frac{p(z|x,\theta)}{q(z)}dz=ELBO+KL(q(z),p(z|x,\theta))
\end{aligned}
{{< /katex >}}
In the equation above, the Evidence Lower Bound (ELBO) is a lower bound, so {{< katex >}}\log p(x|\theta)\ge ELBO{{< /katex >}}. The equality holds when the KL divergence is 0, i.e., {{< katex >}}q(z)=p(z|x,\theta){{< /katex >}}. The purpose of the EM algorithm is to maximize the ELBO. According to the proof process above, the maximum ELBO is obtained at each step of EM, and the parameters that maximize the ELBO are used as input for the next step:

{{< katex >}}
\hat{\theta}=\mathop{argmax}_{\theta}ELBO=\mathop{argmax}_\theta\int_zq(z)\log\frac{p(x,z|\theta)}{q(z)}dz
{{< /katex >}}

Since the maximum value can be achieved when {{< katex >}} q(z)=p(z|x,\theta^t){{< /katex >}}, we have:

{{< katex display=true class="optional" >}}
\begin{aligned}
\hat{\theta}&=\mathop{argmax}_{\theta}ELBO \\
&=\mathop{argmax}_\theta\int_zq(z)\log\frac{p(x,z|\theta)}{q(z)}dz \\
&=\mathop{argmax}_\theta\int_zp(z|x,\theta^t)\log\frac{p(x,z|\theta)}{p(z|x,\theta^t)}dz \\
&=\mathop{argmax}_\theta\int_z p(z|x,\theta^t)\log p(x,z|\theta)
\end{aligned}
{{< /katex >}}
This formula is the one used in the EM iteration process above.

Starting from Jensen\'s inequality, this formula can also be derived:

{{< katex display=true class="optional" >}}
\begin{aligned}
\log p(x|\theta)&=\log\int_zp(x,z|\theta)dz \\
&=\log\int_z\frac{p(x,z|\theta)q(z)}{q(z)}dz \\
&=\log \mathbb{E}_{q(z)}[\frac{p(x,z|\theta)}{q(z)}] \\
&\ge \mathbb{E}_{q(z)}[\log\frac{p(x,z|\theta)}{q(z)}]
\end{aligned}
{{< /katex >}}
In this case, the right side of the equation is the ELBO, and the equality holds when {{< katex >}} p(x,z|\theta)=Cq(z){{< /katex >}}.

Thus: {{< katex >}}
\int_zq(z)dz=\frac{1}{C}\int_zp(x,z|\theta)dz=\frac{1}{C}p(x|\theta)=1\\
\Rightarrow q(z)=\frac{1}{p(x|\theta)}p(x,z|\theta)=p(z|x,\theta)
{{< /katex >}}

We find that this process is the condition for the maximum value to hold as discussed above.

## Generalized EM

The EM model solves the problem of parameter estimation for probabilistic generative models by introducing latent variable {{< katex >}}z{{< /katex >}} to learn {{< katex >}}\theta{{< /katex >}}, and specific models have different assumptions for {{< katex >}}z{{< /katex >}}. For the learning task {{< katex >}}p(x|\theta){{< /katex >}}, it is the learning task {{< katex >}}\frac{p(x,z|\theta)}{p(z|x,\theta)}{{< /katex >}}. In this formula, we assume that in the E step, {{< katex >}}q(z)=p(z|x,\theta){{< /katex >}}. However, if this {{< katex >}}p(z|x,\theta){{< /katex >}} cannot be solved, sampling (MCMC) or variational inference methods must be used to approximate this posterior. We observe the expression of KL divergence. To maximize the ELBO, we need to minimize the KL divergence when {{< katex >}}\theta{{< /katex >}} is fixed:

{{< katex >}}
\hat{q}(z)=\mathop{argmin}_qKL(p,q)=\mathop{argmax}_qELBO
{{< /katex >}}

This is the basic idea of the generalized EM:

1.  E step: {{< katex >}}
      \hat{q}^{t+1}(z)=\mathop{argmax}_q\int_zq^t(z)\log\frac{p(x,z|\theta)}{q^t(z)}dz, \text{ fixed }\theta
      {{< /katex >}}

2.  M step: {{< katex >}}
      \hat{\theta}=\mathop{argmax}_\theta \int_zq^{t+1}(z)\log\frac{p(x,z|\theta)}{q^{t+1}(z)}dz, \text{ fixed }\hat{q}
      {{< /katex >}}

For the integration above: {{< katex >}}
ELBO=\int_zq(z)\log\frac{p(x,z|\theta)}{q(z)}dz=\mathbb{E}_{q(z)}[p(x,z|\theta)]+Entropy(q(z))
{{< /katex >}}

Therefore, we see that the generalized EM is equivalent to adding an entropy term to the original formula.

## Generalization of EM

The EM algorithm is similar to the coordinate ascent method, where some coordinates are fixed and others are optimized, and then iterated repeatedly. If the posterior probability of {{< katex >}}z{{< /katex >}} cannot be solved within the EM framework, some variations of EM need to be adopted to estimate this posterior.

1.  Variational inference based on mean-field, VBEM/VEM
2.  Monte Carlo-based EM, MCEM

# Gaussian Mixture Model

To solve the unimodality problem of the Gaussian model, we introduce the weighted average of multiple Gaussian models to fit the multimodal data:

{{< katex >}}
p(x)=\sum\limits_{k=1}^K\alpha_k\mathcal{N}(\mu_k,\Sigma_k)
{{< /katex >}}

We introduce the latent variable {{< katex >}}z{{< /katex >}}, which represents which Gaussian distribution the corresponding sample {{< katex >}}x{{< /katex >}} belongs to. This variable is a discrete random variable:

{{< katex >}}
p(z=i)=p_i,\sum\limits_{i=1}^kp(z=i)=1
{{< /katex >}}

As a generative model, the Gaussian mixture model generates samples through the distribution of the latent variable {{< katex >}}z{{< /katex >}}. It can be represented with a probability graph:

![](./img/gmm_1d.png){width="30px"}

In this graph, node {{< katex >}}z{{< /katex >}} represents the probability mentioned above, and {{< katex >}}x{{< /katex >}} represents the generated Gaussian distribution.

Therefore, for {{< katex >}}p(x){{< /katex >}}: {{< katex >}}
p(x)=\sum\limits_zp(x,z)=\sum\limits_{k=1}^Kp(x,z=k)=\sum\limits_{k=1}^Kp(z=k)p(x|z=k)
{{< /katex >}}

Thus: {{< katex >}}
p(x)=\sum\limits_{k=1}^Kp_k\mathcal{N}(x|\mu_k,\Sigma_k)
{{< /katex >}}

## MLE

The samples are {{< katex >}}X=(x_1,x_2,\cdots,x_N){{< /katex >}}, and {{< katex >}}(X,Z){{< /katex >}} are the complete parameters, with the parameters being {{< katex >}}\theta=\{p_1,p_2,\cdots,p_K,\mu_1,\mu_2,\cdots,\mu_K\Sigma_1,\Sigma_2,\cdots,\Sigma_K\}{{< /katex >}}

We obtain the value of {{< katex >}}\theta{{< /katex >}} through maximum likelihood estimation:

{{< katex display=true class="optional" >}}
\begin{aligned}\theta_{MLE}&=\mathop{argmax}\limits_{\theta}\log p(X)=\mathop{argmax}_{\theta}\sum\limits_{i=1}^N\log p(x_i)\\
&=\mathop{argmax}_\theta\sum\limits_{i=1}^N\log \sum\limits_{k=1}^Kp_k\mathcal{N}(x_i|\mu_k,\Sigma_k)
\end{aligned}
{{< /katex >}}
This expression cannot be obtained by direct derivation due to the presence of the summation, so the EM algorithm is needed.

## Solve GMM using EM

The basic expression of the EM algorithm is: {{< katex >}}\theta^{t+1}=\mathop{argmax}\limits_{\theta}\mathbb{E}_{z|x,\theta_t}[p(x,z|\theta)]{{< /katex >}}. Applying the GMM expression for the dataset, we get:

{{< katex display=true class="optional" >}}
\begin{aligned}
Q(\theta,\theta^t)&=\sum\limits_z[\log\prod\limits_{i=1}^Np(x_i,z_i|\theta)]\prod \limits_{i=1}^Np(z_i|x_i,\theta^t)\\
&=\sum\limits_z[\sum\limits_{i=1}^N\log p(x_i,z_i|\theta)]\prod \limits_{i=1}^Np(z_i|x_i,\theta^t)
\end{aligned}
{{< /katex >}}

For the summation in the middle, expanding the first term, we have:

{{< katex display=true class="optional" >}}
\begin{aligned}
\sum\limits_z\log p(x_1,z_1|\theta)\prod\limits_{i=1}^Np(z_i|x_i,\theta^t)&=\sum\limits_z\log p(x_1,z_1|\theta)p(z_1|x_1,\theta^t)\prod\limits_{i=2}^Np(z_i|x_i,\theta^t)\\
&=\sum\limits_{z_1}\log p(x_1,z_1|\theta)
p(z_1|x_1,\theta^t)\sum\limits_{z_2,\cdots,z_K}\prod\limits_{i=2}^Np(z_i|x_i,\theta^t)\\
&=\sum\limits_{z_1}\log p(x_1,z_1|\theta)p(z_1|x_1,\theta^t)
\end{aligned}
{{< /katex >}}
Similarly, {{< katex >}}Q{{< /katex >}} can be written as: {{< katex >}}
Q(\theta,\theta^t)=\sum\limits_{i=1}^N\sum\limits_{z_i}\log p(x_i,z_i|\theta)p(z_i|x_i,\theta^t)
{{< /katex >}}

For {{< katex >}}p(x,z|\theta){{< /katex >}}: {{< katex >}}
p(x,z|\theta)=p(z|\theta)p(x|z,\theta)=p_z\mathcal{N}(x|\mu_z,\Sigma_z)
{{< /katex >}}

For {{< katex >}}p(z|x,\theta^t){{< /katex >}}: {{< katex >}}
p(z|x,\theta^t)=\frac{p(x,z|\theta^t)}{p(x|\theta^t)}=\frac{p_z^t\mathcal{N}(x|\mu_z^t,\Sigma_z^t)}{\sum\limits_kp_k^t\mathcal{N}(x|\mu_k^t,\Sigma_k^t)}
{{< /katex >}}

Plug in {{< katex >}}Q{{< /katex >}}:

{{< katex >}}
Q=\sum\limits_{i=1}^N\sum\limits_{z_i}\log p_{z_i}\mathcal{N(x_i|\mu_{z_i},\Sigma_{z_i})}\frac{p_{z_i}^t\mathcal{N}(x_i|\mu_{z_i}^t,\Sigma_{z_i}^t)}{\sum\limits_kp_k^t\mathcal{N}(x_i|\mu_k^t,\Sigma_k^t)}
{{< /katex >}}

Find maximum of {{< katex >}}Q{{< /katex >}}: {{< katex >}}
Q=\sum\limits_{k=1}^K\sum\limits_{i=1}^N[\log p_k+\log \mathcal{N}(x_i|\mu_k,\Sigma_k)]p(z_i=k|x_i,\theta^t)
{{< /katex >}}

1.  {{< katex >}}p_k^{t+1}{{< /katex >}}:

    {{< katex >}}
      p_k^{t+1}=\mathop{argmax}_{p_k}\sum\limits_{k=1}^K\sum\limits_{i=1}^N[\log p_k+\log \mathcal{N}(x_i|\mu_k,\Sigma_k)]p(z_i=k|x_i,\theta^t)\ s.t.\ \sum\limits_{k=1}^Kp_k=1
      {{< /katex >}}

    {{< katex >}}\therefore\\
      p_k^{t+1}=\mathop{argmax}_{p_k}\sum\limits_{k=1}^K\sum\limits_{i=1}^N\log p_kp(z_i=k|x_i,\theta^t)\ s.t.\ \sum\limits_{k=1}^Kp_k=1
      {{< /katex >}}

    Introduce a Lagrange multiplier:

    {{< katex >}}L(p_k,\lambda)=\sum\limits_{k=1}^K\sum\limits_{i=1}^N\log p_kp(z_i=k|x_i,\theta^t)-\lambda(1-\sum\limits_{k=1}^Kp_k){{< /katex >}}

    {{< katex >}}\therefore\\
      \frac{\partial}{\partial p_k}L=\sum\limits_{i=1}^N\frac{1}{p_k}p(z_i=k|x_i,\theta^t)+\lambda=0\\
      \Rightarrow \sum\limits_k\sum\limits_{i=1}^N\frac{1}{p_k}p(z_i=k|x_i,\theta^t)+\lambda\sum\limits_kp_k=0\\
      \Rightarrow\lambda=-N
      {{< /katex >}}

    {{< katex >}}\therefore\\
      p_k^{t+1}=\frac{1}{N}\sum\limits_{i=1}^Np(z_i=k|x_i,\theta^t)
      {{< /katex >}}

2.  {{< katex >}}\mu_k,\Sigma_k{{< /katex >}}, these two parameters are unconstrained, and can be directly derived.
